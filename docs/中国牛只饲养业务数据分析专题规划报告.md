## 专题一：牛只生长绩效分析

### 1. 专题背景与业务价值

在中国牛只饲养行业中，**牛只生长绩效分析**是实现精细化管理和提升经济效益的关键环节。随着畜牧业规模化、集约化发展，传统依靠经验判断的饲养模式已难以适应市场竞争和成本控制的需求。行业痛点主要体现在：牛只个体生长差异大，难以统一管理，导致饲料浪费和出栏时间不确定；缺乏科学的增重预测和料肉比评估，使得饲养周期过长或过短，影响肉品品质和市场竞争力；疾病预警和健康管理不足，疫病发生率高，造成重大经济损失。

数据驱动的价值在于，通过对牛只生长相关数据的全面采集、深度分析与智能预测，可以实现对牛只个体和群体的精准管理。具体而言，能够帮助养殖企业优化饲养策略（根据牛只的生长曲线和营养需求，动态调整饲料配方和投喂方案，提高饲料转化效率）、缩短饲养周期（通过精准预测牛只的增重速度和出栏体重，合理规划出栏时间）、提升肉品品质（科学的饲养管理有助于改善牛只健康状况，提高肉质）、降低运营风险（通过对生长异常的早期预警，及时发现并干预潜在问题）以及辅助经营决策（为牧场管理者提供数据支持，优化种群结构、采购策略和销售计划）。

### 2. 核心分析目标

本专题的核心分析目标是回答以下关键业务问题：

1. 牛只个体日增重（ADG）的波动规律及影响因素是什么？
2. 如何准确预测牛只的未来增重趋势和达到目标出栏体重所需时间？
3. 不同饲料配方和投喂方案对料肉比（FCR）的影响效果如何？
4. 哪些牛只个体或群体存在生长迟缓或异常增重，其原因可能是什么？
5. 如何通过数据分析优化牛只的饲养周期，实现最佳出栏效益？
6. 环境因素（如温度、湿度、季节）对牛只生长绩效有何显著影响？
7. 疾病或健康状况对牛只生长绩效的影响程度如何？

### 3. 数据来源与关键字段

| 数据域 | 主要数据表 | 关键字段 |
|-------|----------|---------|
| 资产/牛只域 | `ranch_onstall`, `psi_cattle_grow_config` | cattle_id, install_date, weight, latest_weight, out_stall_weight |
| 饲养/投喂域 | `psi_cattle_feed_detail`, `psi_recipe`, `psi_stall_feeding_plan` | feed_date, feed_amount, recipe_id, commodity_id |
| 监控/巡检域 | `psi_stall_ai_data`, `psi_cattle_ai_score_result` | weight, score, hair, muscle, act_count |
| 疫病/健康域 | `psi_ranch_sick_cattle` | sick_date, status, cure_date |
| 基础主数据域 | `ranch`, `ranch_stall` | ranch_id, stall_id, recipe_id, system_cattle_num |

### 4. 核心分析指标体系

| 一级指标 | 二级指标 | 计算逻辑 |
|---------|---------|---------|
| **生长效率** | 日均增重（ADG） | (当前体重 - 上次称重体重) / 间隔天数 |
| **生长效率** | 料肉比（FCR） | 总采食量 / 总增重 |
| **生长效率** | 出栏体重达标率 | 达标牛只数 / 总出栏牛只数 |
| **生长预测** | 预计出栏日期 | 基于当前ADG和目标体重的预测模型输出 |
| **生长预测** | 生长曲线拟合度 | 实际生长曲线与标准曲线的R²值 |
| **健康关联** | 疾病对增重的影响系数 | 患病期间ADG / 健康期间ADG |
| **环境关联** | 温度-增重相关系数 | Pearson相关系数（温度 vs ADG） |
| **经济效益** | 单头牛饲养成本 | 饲料成本 + 药品成本 + 服务费 |
| **经济效益** | 单位增重成本 | 总饲养成本 / 总增重量 |
| **群体对比** | 栏舍间ADG差异系数 | 各栏舍ADG的变异系数（CV） |

### 5. 分析方法与模型

本专题推荐以下分析方法：**时间序列分析**（ARIMA/LSTM）用于牛只体重增长趋势预测；**回归分析**（多元线性回归、随机森林回归）用于料肉比影响因素分析；**聚类分析**（K-Means）用于牛只生长表现分群；**异常检测**（IQR法、Isolation Forest）用于识别生长异常个体；**生存分析**用于出栏时机优化建模。

### 6. 分析维度与切片

分析维度涵盖**时间维度**（日/周/月/季度/年度趋势，入栏天数分段）、**空间维度**（牧场级、栏舍级、区域级对比）、**个体维度**（品种、性别、入栏体重区间、健康状态）以及**饲料维度**（配方类型、TMR方案、投喂量区间）。

### 7. 数据产品形态

本专题应呈现为：**实时生长监控看板**（含当前在栏牛只ADG分布热力图、料肉比趋势折线图）、**月度生长绩效报告**（含栏舍对比柱状图、生长曲线拟合图）、**出栏预测工具**（输入当前体重，输出预计出栏日期和收益估算）以及**生长异常预警推送**（当个体ADG连续3天低于阈值时自动触发）。

### 8. 实施路径与优先级

**优先级：高**。建议第一阶段（1-2周）完成数据清洗和基础指标计算；第二阶段（3-4周）构建生长预测模型；第三阶段（5-6周）开发数据产品并上线。预期难点在于历史体重数据的完整性，解决思路是利用AI评分数据（`psi_cattle_ai_score_result`）中的体重字段进行补充。

## 专题二：供应链金融风控分析

### 1. 专题背景与业务价值

中国牛只饲养行业面临融资难、融资贵的突出问题。以活体牛只为质押物的供应链金融模式，是解决养殖企业资金需求的创新路径，但同时也带来了独特的风险管理挑战。活体资产具有价值波动性大、难以标准化评估、存在自然损耗等特点，使得传统金融风控手段难以直接适用。

数据驱动的风控体系能够实现对质押资产的实时监控和动态评估，通过整合牛只生长数据、健康数据、AI监控数据和贷款数据，构建多维度的风险预警模型，有效防范贷款违约风险，保障金融机构和养殖企业的双方利益。

### 2. 核心分析目标

1. 当前贷款组合的整体质押率水平如何，是否存在质押不足的风险敞口？
2. 哪些牧场或借款人的贷款风险较高，风险集中在哪些维度？
3. 牛只健康状况和生长异常对质押资产价值的影响程度如何？
4. 还款进度是否符合预期，逾期风险的早期信号有哪些？
5. 活体资产的价值评估模型如何构建，如何实现动态更新？
6. 如何通过AI监控数据验证质押牛只的真实在栏状态？
7. 不同还款方式（按重增信、出栏还款）的风险特征有何差异？

### 3. 数据来源与关键字段

| 数据域 | 主要数据表 | 关键字段 |
|-------|----------|---------|
| 贷款/金融域 | `psi_loan_plan`, `psi_loan_repay`, `ranch_loan` | loan_amount, cattle_num, contract_no, status, amount, loan_balance |
| 资产/牛只域 | `ranch_onstall`, `psi_cattle_ai_score_result` | weight, price, is_loan, latest_weight, score |
| 监控/巡检域 | `psi_ranch_patrol`, `psi_stall_ai_data`, `psi_warning_record` | first_score, second_score, cattle_count, value |
| 疫病/健康域 | `psi_ranch_sick_cattle` | sick_date, status |
| 基础主数据域 | `psi_customer`, `ranch` | customer_type, status |

### 4. 核心分析指标体系

| 一级指标 | 二级指标 | 计算逻辑 |
|---------|---------|---------|
| **质押覆盖率** | 实时质押率 | 质押牛只当前市值 / 贷款余额 |
| **质押覆盖率** | 质押率预警阈值触发率 | 质押率低于预设阈值的贷款数 / 总贷款数 |
| **资产健康度** | 质押牛只平均健康评分 | 质押牛只AI评分均值 |
| **资产健康度** | 质押牛只病牛率 | 质押牛只中病牛数 / 质押牛只总数 |
| **还款风险** | 还款计划完成率 | 实际还款额 / 计划还款额 |
| **还款风险** | 逾期贷款比率 | 逾期贷款余额 / 总贷款余额 |
| **资产真实性** | AI盘点在栏率 | AI盘点牛只数 / 贷款合同牛只数 |
| **集中度风险** | 单一借款人贷款集中度 | 最大借款人贷款额 / 总贷款额 |
| **增信效率** | 按重增信还款效率 | 实际增重量 × 单位增重还款额 / 计划还款额 |
| **风险预警** | 综合风险评分 | 加权综合多维度风险指标的复合评分 |
| **贷款到期** | 到期贷款预警数 | 30日内到期且未还清的贷款数 |
| **价值损失** | 质押资产价值损失率 | (合同质押价值 - 当前市值) / 合同质押价值 |

### 5. 分析方法与模型

推荐方法包括：**评分卡模型**（Logistic回归）用于贷款违约概率预测；**动态定价模型**（基于生长曲线和市场价格）用于活体资产实时估值；**蒙特卡洛模拟**用于压力测试和极端情景分析；**图神经网络**用于关联风险识别（借款人-牧场-牛只网络）；**时间序列预测**用于还款能力预测。

### 6. 分析维度与切片

分析维度包括**借款人维度**（单一借款人风险画像）、**牧场维度**（牧场级风险评级）、**时间维度**（贷款期限、还款节点）、**资产维度**（牛只品种、体重区间、健康状态）以及**区域维度**（地区性风险集中度）。

### 7. 数据产品形态

本专题应呈现为：**实时风险监控看板**（含质押率仪表盘、风险热力图）、**贷款风险评级报告**（月度，含风险分层矩阵）、**质押资产价值追踪系统**（日度更新）以及**风险预警推送**（质押率触发阈值时自动通知相关人员）。

### 8. 实施路径与优先级

**优先级：高**。建议第一阶段构建基础质押率计算体系；第二阶段开发动态估值模型；第三阶段集成AI监控数据，构建综合风险评分体系。核心难点在于活体资产市场价格的实时获取，解决思路是建立内部价格基准（基于历史成交数据）并定期与外部市场价格校准。

## 专题三：饲料成本优化分析

### 1. 专题背景与业务价值

饲料成本通常占牛只饲养总成本的60%-70%，是影响牧场盈利能力最关键的成本项。在中国，随着饲料原料价格波动加剧，科学的饲料成本管理已成为牧场竞争力的核心要素。当前行业痛点包括：配方设计缺乏数据支撑，难以在营养需求和成本之间取得最优平衡；TMR搅拌投喂的实际效果缺乏量化评估；采购时机和批量决策依赖经验，难以应对价格波动。

数据驱动的饲料成本优化，能够实现从配方设计、投喂执行到采购管理的全链路优化，帮助牧场在保证牛只生长绩效的前提下，最大限度地降低饲料成本。

### 2. 核心分析目标

1. 当前各饲料配方的实际料肉比表现如何，哪些配方性价比最高？
2. TMR搅拌投喂的实际执行量与计划量的偏差有多大，偏差对生长绩效有何影响？
3. 不同体重阶段的牛只，最优饲料配方组合是什么？
4. 饲料采购成本结构如何，主要成本驱动因素是什么？
5. 如何通过采购时机优化和批量策略，降低饲料原料采购成本？
6. 饲料浪费（剩料、回收）的规模和原因是什么？
7. 不同供应商的饲料质量和价格表现如何？

### 3. 数据来源与关键字段

| 数据域 | 主要数据表 | 关键字段 |
|-------|----------|---------|
| 饲养/投喂域 | `psi_recipe`, `psi_feed_scheme`, `psi_stall_feeding_plan`, `psi_cattle_feed_detail` | commodity_id, weight_begin, weight_end, day_age, planned_amount, actual_amount |
| 采购/库存域 | `psi_purchase`, `psi_purchase_detail`, `psi_warehouse_stock` | commodity_id, quantity, price, total_price, purchase_date |
| 成本/费用域 | `psi_cost_detail`, `psi_daily_consume` | commodity_id, quantity, average_price, consume_date |
| 资产/牛只域 | `ranch_onstall`, `psi_cattle_grow_config` | weight, latest_weight |
| 基础主数据域 | `psi_commodity`, `psi_customer` | code, name, type, brand, spec |

### 4. 核心分析指标体系

| 一级指标 | 二级指标 | 计算逻辑 |
|---------|---------|---------|
| **配方效率** | 配方料肉比 | 使用该配方的总采食量 / 总增重量 |
| **配方效率** | 配方单位增重成本 | 使用该配方的总饲料成本 / 总增重量 |
| **投喂执行** | 投喂计划完成率 | 实际投喂量 / 计划投喂量 |
| **投喂执行** | 剩料率 | 回收饲料量 / 投喂总量 |
| **采购成本** | 饲料采购单价趋势 | 各原料月度均价及同比/环比变化 |
| **采购成本** | 采购价格波动率 | 采购价格标准差 / 均价 |
| **库存管理** | 饲料库存周转天数 | 平均库存量 / 日均消耗量 |
| **库存管理** | 库存积压率 | 超过安全库存上限的商品数 / 总商品数 |
| **成本结构** | 饲料成本占比 | 饲料成本 / 总饲养成本 |
| **成本结构** | 各原料成本占比 | 单一原料成本 / 总饲料成本 |
| **供应商评估** | 供应商价格竞争力指数 | 供应商报价 / 市场均价 |
| **综合效益** | 饲料成本降低潜力 | 当前成本 - 最优配方成本（模拟值） |

### 5. 分析方法与模型

推荐方法包括：**线性规划**（饲料配方最优化，在满足营养约束的前提下最小化成本）；**时间序列预测**（饲料原料价格预测，辅助采购决策）；**ABC分析**（饲料原料成本分类管理）；**回归分析**（投喂量偏差与生长绩效的关系）；**对比实验分析**（不同配方方案的A/B测试）。

### 6. 分析维度与切片

分析维度包括**配方维度**（不同配方的效果对比）、**体重阶段维度**（不同生长阶段的饲料需求）、**时间维度**（季节性饲料成本变化）、**栏舍维度**（不同栏舍的投喂执行质量）以及**供应商维度**（供应商绩效对比）。

### 7. 数据产品形态

本专题应呈现为：**饲料成本分析看板**（含成本结构饼图、料肉比趋势图）、**配方效果评估报告**（月度，含配方对比矩阵）、**采购决策辅助工具**（基于价格预测的最优采购时机建议）以及**投喂异常预警**（当实际投喂量偏差超过阈值时触发）。

### 8. 实施路径与优先级

**优先级：高**。建议第一阶段建立饲料成本基准数据集；第二阶段开发配方效果评估模型；第三阶段构建采购优化工具。核心难点在于投喂明细数据的颗粒度和准确性，解决思路是通过TMR搅拌车记录（`psi_tmr`）和栏舍投喂明细（`psi_stall_feed_detail`）进行交叉验证。

## 专题四：疫病健康管理分析

### 1. 专题背景与业务价值

牛只疫病是影响牧场经济效益的重要风险因素。在中国，规模化养殖场面临的主要疫病风险包括口蹄疫、牛病毒性腹泻、呼吸道疾病综合征等，一旦暴发可能造成重大经济损失。同时，亚临床疾病（症状不明显但影响生长绩效）的识别和管理也是精细化养殖的重要课题。

数据驱动的健康管理体系，能够通过对病牛档案、治疗记录、免疫记录和巡检数据的综合分析，实现疫病的早期预警、精准诊断和科学防控，将被动应对转变为主动预防，显著降低疫病损失。

### 2. 核心分析目标

1. 牛只发病率的时间趋势和季节性规律是什么？
2. 各类疾病的发病率、治愈率和死亡率分别是多少？
3. 哪些牛只个体或群体是高风险群体（发病概率更高）？
4. 不同治疗方案的效果和成本效益如何？
5. 免疫接种和驱虫计划的覆盖率和执行质量如何？
6. 疾病对牛只生长绩效的量化影响是多少？
7. 如何构建疾病早期预警模型，在临床症状出现前识别高风险个体？

### 3. 数据来源与关键字段

| 数据域 | 主要数据表 | 关键字段 |
|-------|----------|---------|
| 疫病/健康域 | `psi_ranch_sick_cattle`, `psi_ranch_sick_cattle_cure`, `psi_cure_scheme`, `psi_immune_guard`, `psi_insect_repellent_plan` | sick_date, status, cure_date, symptoms, cure_price, disease_code |
| 资产/牛只域 | `ranch_onstall`, `psi_cattle_ai_score_result` | weight, latest_weight, score, hair, muscle |
| 监控/巡检域 | `psi_ranch_patrol`, `psi_ai_alert_record` | first_score, second_score, slot_ratio, type, urgency_level |
| 成本/费用域 | `psi_cost_detail` | commodity_id, quantity, average_price |
| 基础主数据域 | `psi_commodity` | type（药品类型） |

### 4. 核心分析指标体系

| 一级指标 | 二级指标 | 计算逻辑 |
|---------|---------|---------|
| **发病状况** | 月度发病率 | 当月新发病牛只数 / 月初在栏牛只数 |
| **发病状况** | 疾病类型分布 | 各疾病类型发病数 / 总发病数 |
| **治疗效果** | 治愈率 | 治愈牛只数 / 总发病牛只数 |
| **治疗效果** | 平均治疗周期 | 治愈日期 - 发病日期的均值 |
| **治疗成本** | 单头牛治疗成本 | 治疗用药费用 + 兽医服务费 |
| **治疗成本** | 疾病总损失 | 治疗成本 + 生长损失（患病期间少增重 × 单位增重价值） |
| **预防管理** | 免疫接种覆盖率 | 已接种牛只数 / 应接种牛只数 |
| **预防管理** | 驱虫计划执行率 | 实际驱虫牛只数 / 计划驱虫牛只数 |
| **健康评估** | 牛只平均体况评分 | AI评分中体况评分的均值 |
| **风险预警** | 高风险个体识别率 | 预警模型识别出的高风险牛只数 / 实际发病牛只数（召回率） |

### 5. 分析方法与模型

推荐方法包括：**生存分析**（Kaplan-Meier曲线）用于疾病持续时间和治愈概率分析；**逻辑回归/随机森林**用于疾病发生概率预测（早期预警模型）；**聚类分析**用于疾病模式识别和高风险群体划分；**成本效益分析**用于不同治疗方案的经济学评估；**时间序列分析**用于疾病季节性规律挖掘。

### 6. 分析维度与切片

分析维度包括**时间维度**（月度/季度发病趋势，季节性分析）、**疾病维度**（疾病类型、严重程度分级）、**个体维度**（品种、年龄、体重区间、历史健康记录）、**空间维度**（牧场级、栏舍级疾病热点）以及**治疗维度**（治疗方案、用药类型、兽医绩效）。

### 7. 数据产品形态

本专题应呈现为：**健康管理看板**（含在栏牛只健康状态分布、发病率趋势折线图）、**疾病预警系统**（基于AI评分和巡检数据的早期预警）、**月度健康报告**（含疾病类型分布、治疗效果评估）以及**免疫驱虫计划执行追踪**（日历视图，显示计划执行进度）。

### 8. 实施路径与优先级

**优先级：高**。建议第一阶段建立疾病基础统计体系；第二阶段开发疾病预警模型；第三阶段集成AI评分数据，构建综合健康评估体系。核心难点在于历史病牛数据的标准化（疾病名称、症状描述的规范化），解决思路是建立标准化的疾病编码体系（参考`psi_cattle_disease_type`表）。

## 专题五：栏舍运营效率分析

### 1. 专题背景与业务价值

栏舍是牧场最核心的固定资产，其运营效率直接决定了牧场的整体产出和盈利能力。在中国规模化养殖背景下，栏舍资源的优化配置是提升牧场竞争力的重要手段。当前行业痛点包括：栏舍容量利用率参差不齐，部分栏舍长期空置或过度拥挤；不同栏舍的生长绩效差异显著，但缺乏系统性的对标分析；栏舍周转率低，影响资产回报率。

数据驱动的栏舍运营效率分析，能够帮助牧场管理者全面了解各栏舍的运营状态，识别高效和低效栏舍，挖掘改进潜力，优化栏舍资源配置策略。

### 2. 核心分析目标

1. 各栏舍的容量利用率如何，是否存在明显的过度利用或利用不足？
2. 不同栏舍的牛只生长绩效（ADG、FCR）有何差异，影响因素是什么？
3. 栏舍的牛只周转率如何，如何优化入栏和出栏的节奏？
4. 栏舍配方（`recipe_id`）与生长绩效的关联性如何？
5. 哪些栏舍是"明星栏舍"（高绩效），哪些是"问题栏舍"（低绩效）？
6. 栏舍的历史绩效趋势如何，是否存在系统性的改善或恶化？
7. 如何根据牛只特征（体重、品种、健康状态）优化栏舍分配策略？

### 3. 数据来源与关键字段

| 数据域 | 主要数据表 | 关键字段 |
|-------|----------|---------|
| 基础主数据域 | `ranch_stall` | name, recipe_id, system_cattle_num, weight |
| 资产/牛只域 | `ranch_onstall`, `psi_stall_ai_data` | install_date, weight, latest_weight, stall_id, cattle_count, value |
| 饲养/投喂域 | `psi_stall_feeding_plan`, `psi_stall_feed_detail` | stall_id, plan_date, actual_amount |
| 监控/巡检域 | `psi_ranch_patrol`, `psi_stall_daily_report` | stall_id, first_score, second_score, slot_ratio |
| 成本/费用域 | `psi_cost_detail`, `psi_cost_share` | stall_id, commodity_id, quantity, average_price |

### 4. 核心分析指标体系

| 一级指标 | 二级指标 | 计算逻辑 |
|---------|---------|---------|
| **容量利用** | 栏舍容量利用率 | 实际在栏牛只数 / 设计容量（system_cattle_num） |
| **容量利用** | 空栏率 | 空栏天数 / 总天数 |
| **生长绩效** | 栏舍平均ADG | 栏舍内所有牛只ADG的均值 |
| **生长绩效** | 栏舍料肉比 | 栏舍总投喂量 / 栏舍总增重量 |
| **周转效率** | 栏舍牛只周转率 | 出栏牛只数 / 平均在栏牛只数（年化） |
| **周转效率** | 平均在栏天数 | 出栏牛只的平均在栏时长 |
| **成本效率** | 栏舍单位面积产出 | 栏舍总增重量 / 栏舍面积 |
| **成本效率** | 栏舍单头牛成本 | 栏舍总成本 / 平均在栏牛只数 |
| **健康状况** | 栏舍病牛率 | 栏舍病牛数 / 在栏牛只数 |
| **巡检质量** | 栏舍巡检评分 | 巡检记录中first_score和second_score的均值 |
| **综合绩效** | 栏舍综合绩效指数 | 加权综合多维度指标的复合评分 |

### 5. 分析方法与模型

推荐方法包括：**数据包络分析（DEA）**用于栏舍运营效率的相对评估；**聚类分析**用于栏舍绩效分群（明星栏舍、问题栏舍、普通栏舍）；**回归分析**用于栏舍绩效影响因素识别；**时间序列分析**用于栏舍绩效趋势分析；**仿真模拟**用于栏舍分配策略优化。

### 6. 分析维度与切片

分析维度包括**空间维度**（单栏舍、牧场内横向对比、跨牧场对比）、**时间维度**（月度/季度绩效趋势）、**配方维度**（不同饲料配方对栏舍绩效的影响）以及**牛只特征维度**（不同体重区间、品种的栏舍绩效差异）。

### 7. 数据产品形态

本专题应呈现为：**栏舍绩效看板**（含容量利用率热力图、绩效排名榜）、**月度栏舍绩效报告**（含栏舍对比矩阵、趋势分析）以及**栏舍分配建议工具**（根据牛只特征推荐最优栏舍）。

### 8. 实施路径与优先级

**优先级：高**。建议第一阶段建立栏舍基础绩效指标体系；第二阶段开发栏舍综合绩效评分模型；第三阶段构建栏舍分配优化工具。核心难点在于栏舍成本的准确分摊，解决思路是利用`psi_cost_share`表的成本分摊数据，建立合理的成本归因模型。

## 专题六：AI智能监控预警分析

### 1. 专题背景与业务价值

本牧场业务系统已建立了较为完善的AI智能监控体系，包括视频监控、AI牛只识别、异常行为检测和自动预警机制。然而，AI系统的实际效能如何、预警准确率是否达到预期、响应时效是否满足业务需求，这些问题需要通过数据分析来客观评估。

通过对AI监控系统效能的系统性分析，可以发现系统的薄弱环节，指导技术优化方向，最终实现AI监控系统价值的最大化，为牧场的智能化管理提供坚实支撑。

### 2. 核心分析目标

1. AI预警的准确率（精确率和召回率）如何，误报率和漏报率分别是多少？
2. 预警从触发到处理完成的平均时效如何，是否存在响应延迟的问题？
3. 不同类型的AI预警（异常行为、设备离线、牛只异常等）的发生频率和处理效果如何？
4. AI盘点数据与人工盘点数据的一致性如何，差异主要出现在哪些场景？
5. 哪些栏舍或时间段的预警频率异常高，背后的原因是什么？
6. AI监控系统的设备在线率和数据质量如何？
7. AI预警对牛只健康和安全事件的预防效果如何（与历史基准对比）？

### 3. 数据来源与关键字段

| 数据域 | 主要数据表 | 关键字段 |
|-------|----------|---------|
| 监控/巡检域 | `psi_ai_alert_record`, `psi_warning_record`, `psi_ranch_patrol`, `psi_stall_ai_data`, `psi_cattle_num_check_log` | type, urgency_level, event_desc, trigger_time, warning_num, cattle_count |
| 基础主数据域 | `psi_monitor_device` | device_name, device_serial_no, stall_id, media_url |
| 计划/任务域 | `psi_work_order`, `psi_task_info` | work_order_status, step_name |
| 资产/牛只域 | `psi_stall_ai_data`, `psi_cattle_ai_score_result` | cattle_count, value, act_count, score |

### 4. 核心分析指标体系

| 一级指标 | 二级指标 | 计算逻辑 |
|---------|---------|---------|
| **预警准确性** | 预警精确率 | 真正预警数 / (真正预警数 + 误报数) |
| **预警准确性** | 预警召回率 | 真正预警数 / (真正预警数 + 漏报数) |
| **响应时效** | 平均预警响应时间 | 工单处理完成时间 - 预警触发时间的均值 |
| **响应时效** | 预警超时率 | 超过SLA时限的预警数 / 总预警数 |
| **设备状态** | 设备在线率 | 在线设备数 / 总设备数 |
| **设备状态** | 设备故障频率 | 月度设备故障次数 / 设备总数 |
| **盘点准确性** | AI盘点与人工盘点差异率 | |AI盘点数 - 人工盘点数| / 人工盘点数 |
| **预警分布** | 各类型预警占比 | 单一类型预警数 / 总预警数 |
| **处置效果** | 预警关闭率 | 已关闭预警数 / 总预警数 |
| **系统价值** | 预警拦截事件数 | 因预警及时处置而避免的损失事件数 |
| **综合效能** | AI监控系统综合效能评分 | 加权综合多维度指标的复合评分 |

### 5. 分析方法与模型

推荐方法包括：**混淆矩阵分析**用于预警准确率评估；**ROC/AUC分析**用于预警阈值优化；**队列分析**用于预警响应时效分析；**时间序列分析**用于预警频率趋势和异常时段识别；**关联规则挖掘**用于发现预警事件的共现模式。

### 6. 分析维度与切片

分析维度包括**预警类型维度**（不同类型预警的效能对比）、**设备维度**（不同设备的数据质量）、**时间维度**（预警频率的时间分布，如昼夜差异）、**空间维度**（高预警频率栏舍的地理分布）以及**处置人员维度**（不同工作人员的响应时效对比）。

### 7. 数据产品形态

本专题应呈现为：**AI监控效能看板**（含预警准确率仪表盘、响应时效分布图）、**月度AI系统评估报告**（含效能趋势分析、改进建议）以及**设备健康监控看板**（设备在线率、故障率实时显示）。

### 8. 实施路径与优先级

**优先级：高**。建议第一阶段建立预警数据的标注体系（区分真正预警和误报）；第二阶段开发效能评估指标体系；第三阶段构建AI系统优化建议模型。核心难点在于预警真实性的判断（区分真正预警和误报），解决思路是建立人工复核机制，对预警处置结果进行标注，形成训练数据集。

## 专题七：牛只采购与销售决策分析

### 1. 专题背景与业务价值

牛只的采购和销售决策直接决定了牧场的盈利空间。在中国牛肉市场，牛只价格受季节性、区域性、品种等多重因素影响，波动较大。科学的采购决策能够降低入栏成本，而精准的出栏时机选择则能最大化销售收益。当前行业痛点包括：采购来源单一，缺乏供应商评估体系；出栏时机依赖经验判断，难以把握市场价格高点；缺乏对历史采购和销售数据的系统性分析。

数据驱动的采购与销售决策分析，能够帮助牧场建立科学的买卖决策体系，在市场波动中把握最优时机，提升整体盈利能力。

### 2. 核心分析目标

1. 不同采购来源（供应商、地区）的牛只质量（入栏体重、后续生长绩效）如何？

3. 不同入栏体重区间的牛只，其后续生长绩效和盈利能力有何差异？
4. 出栏体重与销售价格的关系如何，最优出栏体重区间是什么？
5. 历史销售价格的趋势和季节性规律是什么，最优出栏时机如何判断？
6. 退回牛只的主要原因是什么，如何在采购环节降低退回率？
7. 如何构建牛只采购质量评估模型，辅助采购决策？

### 3. 数据来源与关键字段

| 数据域 | 主要数据表 | 关键字段 |
|-------|----------|---------|
| 资产/牛只域 | `psi_cattle_purchase`, `psi_cattle_sell`, `psi_cattle_return`, `psi_cattle_price` | code, customer_id, quantity, purchase_cost, sell_cost, return_weight, return_price, reason |
| 资产/牛只域 | `ranch_onstall`, `psi_cattle_grow_config` | weight, latest_weight, install_date, price |
| 基础主数据域 | `psi_customer`, `psi_region` | customer_type, customer_name |
| 报表/统计域 | `psi_cattle_month_report` | commodity_name, start_weight, end_weight, count |

### 4. 核心分析指标体系

| 一级指标 | 二级指标 | 计算逻辑 |
|---------|---------|---------|
| **采购质量** | 供应商牛只平均入栏体重 | 按供应商分组的入栏体重均值 |
| **采购质量** | 供应商牛只后续ADG | 按供应商分组的后续平均日增重 |
| **采购成本** | 采购单价趋势 | 月度采购均价及同比/环比变化 |
| **采购成本** | 单位体重采购成本 | 总采购成本 / 总采购体重 |
| **销售效益** | 出栏溢价率 | (出栏销售价 - 入栏采购价) / 入栏采购价 |
| **销售效益** | 单头牛净利润 | 销售收入 - 采购成本 - 饲养成本 |
| **退回管理** | 退回率 | 退回牛只数 / 采购牛只数 |
| **退回管理** | 退回原因分布 | 各退回原因的占比 |
| **价格敏感性** | 采购价格弹性 | 价格变动1%对应的采购量变动% |
| **时机把握** | 最优出栏体重区间 | 历史数据中单位增重利润最高的体重区间 |
| **周期分析** | 采购-出栏周期 | 出栏日期 - 入栏日期的均值 |
| **综合回报** | 投资回报率（ROI） | 净利润 / 采购成本 |

### 5. 分析方法与模型

推荐方法包括：**时间序列分析**（SARIMA）用于价格季节性规律挖掘；**回归分析**用于出栏体重与销售价格关系建模；**决策树/随机森林**用于采购质量预测模型；**最优化模型**用于出栏时机选择（综合生长预测和价格预测）；**供应商评分卡**用于采购来源评估。

### 6. 分析维度与切片

分析维度包括**供应商维度**（不同供应商的质量和价格对比）、**地区维度**（不同采购地区的牛只质量差异）、**体重维度**（不同入栏/出栏体重区间的盈利分析）、**时间维度**（季节性价格规律）以及**品种维度**（不同品种的生长绩效和价格差异）。

### 7. 数据产品形态

本专题应呈现为：**采购决策看板**（含供应商评分排名、采购价格趋势图）、**出栏决策辅助工具**（输入当前体重和市场价格，输出最优出栏时机建议）以及**月度采购与销售分析报告**（含盈利分析、供应商评估）。

### 8. 实施路径与优先级

**优先级：高**。建议第一阶段建立采购和销售数据的历史基准；第二阶段开发价格预测模型；第三阶段构建综合决策支持工具。核心难点在于外部市场价格数据的获取，解决思路是通过爬取公开市场价格数据（如农业农村部价格监测数据）与内部数据融合。

## 专题八：运营任务与工单管理分析

### 1. 专题背景与业务价值

牧场的日常运营涉及大量的任务计划、工单执行和审批流程，运营管理效率直接影响牧场的整体运作质量。在规模化养殖场中，人员管理和任务协同是重要的管理挑战。数据驱动的运营管理分析，能够帮助管理者全面了解任务执行状况，识别效率瓶颈，优化人员配置，提升整体运营管理水平。

### 2. 核心分析目标

1. 各类任务和工单的完成率、及时完成率如何？
2. 工单从创建到关闭的平均流转时间是多少，哪些环节存在瓶颈？
3. 不同人员的任务完成效率和质量如何，是否存在明显的绩效差异？
4. 审批流程的平均时长和通过率如何，是否存在审批积压问题？
5. 预警触发的工单与主动计划的工单，其处理效率有何差异？
6. 任务执行质量（如巡检评分、投喂执行率）与业务结果（如牛只健康、生长绩效）的关联性如何？
7. 如何通过数据分析优化任务分配和人员排班？

### 3. 数据来源与关键字段

| 数据域 | 主要数据表 | 关键字段 |
|-------|----------|---------|
| 计划/任务域 | `psi_ranch_plan`, `psi_task_info`, `psi_work_order`, `psi_approve_record` | ranch_id, stall_id, type, plan_date, status, work_order_no, work_order_status, step_name, approve_type, approve_time |
| 监控/巡检域 | `psi_ranch_patrol`, `psi_ranch_patrol_daily_rpt` | stall_id, first_score, second_score, slot_ratio |
| 饲养/投喂域 | `psi_stall_feeding_plan`, `psi_feed_plan_daily` | stall_id, plan_date, status |

### 4. 核心分析指标体系

| 一级指标 | 二级指标 | 计算逻辑 |
|---------|---------|---------|
| **任务完成** | 任务整体完成率 | 已完成任务数 / 总任务数 |
| **任务完成** | 任务及时完成率 | 在截止日期前完成的任务数 / 总任务数 |
| **工单效率** | 工单平均处理时长 | (工单关闭时间 - 工单创建时间)的均值 |
| **工单效率** | 工单积压数 | 超过SLA时限未关闭的工单数 |
| **人员绩效** | 人均任务完成数 | 总完成任务数 / 执行人员数 |
| **人员绩效** | 个人任务质量评分 | 基于任务结果质量的综合评分 |
| **审批效率** | 平均审批时长 | (审批完成时间 - 审批提交时间)的均值 |
| **审批效率** | 审批通过率 | 审批通过数 / 总审批数 |
| **执行质量** | 巡检任务评分均值 | 巡检记录中评分的均值 |
| **执行质量** | 投喂计划执行率 | 实际投喂量 / 计划投喂量 |

### 5. 分析方法与模型

推荐方法包括：**流程挖掘（Process Mining）**用于工单流转路径分析和瓶颈识别；**队列分析**用于工单处理时效分析；**回归分析**用于任务执行质量与业务结果的关联分析；**人员绩效评分模型**用于综合评估员工工作表现；**仿真模拟**用于优化任务分配和排班方案。

### 6. 分析维度与切片

分析维度包括**人员维度**（个人绩效对比）、**任务类型维度**（不同类型任务的完成效率）、**时间维度**（工作日/周末、班次差异）、**空间维度**（不同牧场/栏舍的任务执行质量）以及**优先级维度**（紧急/普通任务的处理差异）。

### 7. 数据产品形态

本专题应呈现为：**运营管理看板**（含任务完成率仪表盘、工单积压趋势图）、**人员绩效报告**（月度，含绩效排名和改进建议）以及**工单流程优化报告**（季度，含流程瓶颈分析和优化建议）。

### 8. 实施路径与优先级

**优先级：高**。建议第一阶段建立任务和工单的基础统计体系；第二阶段开发人员绩效评估模型；第三阶段引入流程挖掘技术，深度分析工单流转效率。核心难点在于任务质量的量化评估，解决思路是将任务执行结果与业务指标（如巡检评分、投喂执行率）关联，建立质量评分体系。

## 专题九：牧场综合运营对标分析

### 1. 专题背景与业务价值

对于管理多个牧场的企业而言，跨牧场的横向对比和行业基准对标是发现最佳实践、推动持续改进的重要管理工具。通过综合运营对标分析，管理层可以快速识别哪些牧场运营卓越、哪些需要重点关注，从而实现资源的精准配置和管理能力的整体提升。

### 2. 核心分析目标

1. 各牧场在核心KPI（ADG、FCR、病牛率、质押率等）上的表现如何，排名如何？
2. 哪些牧场是综合运营的标杆，其成功经验可以复制推广？
3. 各牧场的综合运营评分如何，各维度的强项和弱项是什么？
4. 与行业基准相比，牧场整体运营水平处于什么位置？
5. 牧场运营绩效的时间趋势如何，是否呈现持续改善的态势？
6. 不同规模、不同地区的牧场，其运营绩效有何规律性差异？
7. 如何构建牧场综合运营评分体系，支持管理层的资源配置决策？

### 3. 数据来源与关键字段

本专题需要整合所有业务域的汇总数据，主要依赖报表/统计域的聚合数据，包括`psi_cattle_month_report`、`psi_ranch_synthesize`、`psi_stall_daily_report`等，同时需要从各业务域的明细数据中提取牧场级汇总指标。

### 4. 核心分析指标体系

| 维度 | 核心KPI | 权重建议 |
|-----|--------|---------|
| **资产管理** | 在栏牛只数、周转率、出栏达标率 | 25% |
| **生长绩效** | 平均ADG、料肉比、出栏体重达标率 | 25% |
| **健康管理** | 病牛率、治愈率、免疫覆盖率 | 20% |
| **成本控制** | 单头牛饲养成本、饲料成本占比 | 15% |
| **风险管理** | 质押率、贷款逾期率 | 10% |
| **运营效率** | 任务完成率、工单及时率、巡检评分 | 5% |

### 5. 分析方法与模型

推荐方法包括：**平衡计分卡（BSC）**框架用于综合运营评分体系设计；**数据包络分析（DEA）**用于牧场相对效率评估；**雷达图分析**用于多维度绩效可视化；**聚类分析**用于牧场分群（标杆牧场、成长牧场、问题牧场）；**基准分析**用于与行业标准对比。

### 6. 分析维度与切片

分析维度包括**牧场维度**（单牧场画像、跨牧场对比）、**时间维度**（月度/季度/年度趋势）、**规模维度**（大/中/小规模牧场的绩效规律）以及**地区维度**（不同地区牧场的运营差异）。

### 7. 数据产品形态

本专题应呈现为：**管理层综合运营看板**（含牧场排名、综合评分雷达图）、**季度牧场对标报告**（含标杆分析、改进建议）以及**牧场健康度评估报告**（年度，含综合评级和战略建议）。

### 8. 实施路径与优先级

**优先级：高**。建议第一阶段建立统一的牧场级KPI计算口径；第二阶段开发综合评分模型；第三阶段构建管理层看板和对标报告体系。核心难点在于不同牧场数据口径的统一，解决思路是在数据仓库的DIM层建立标准化的牧场维度表，确保跨牧场数据的可比性。

## 专题十：数据质量与AI盘点准确率分析

### 1. 专题背景与业务价值

数据质量是所有数据分析和AI应用的基础。在牧场业务中，数据质量问题可能导致资产评估失真、风控判断失误、运营决策偏差等严重后果。特别是AI牛只盘点作为资产核查的重要手段，其准确率直接关系到质押资产的真实性验证和金融风控的可靠性。

通过系统性的数据质量分析，可以识别数据问题的根源，建立数据质量治理机制，同时通过AI盘点准确率分析，持续优化AI识别算法，提升资产管理的可靠性。

### 2. 核心分析目标

1. 各业务域的数据完整性、准确性、一致性和及时性如何？
2. 关键业务字段（如牛只体重、在栏状态）的数据缺失率和异常值比例是多少？
3. AI牛只盘点的准确率（与人工盘点对比）在不同条件下的表现如何？
4. AI盘点与人工盘点的差异主要出现在哪些场景（时间、栏舍、牛只密度等）？
5. 数据录入的及时性如何，是否存在大量滞后录入的情况？
6. 跨系统数据的一致性如何（如采购数据与库存数据的匹配度）？
7. 如何建立持续的数据质量监控机制？

### 3. 数据来源与关键字段

本专题需要对所有业务域的数据进行质量评估，重点关注以下数据表：`ranch_onstall`（在栏牛只核心数据）、`psi_cattle_num_check_log`（AI盘点日志）、`psi_cattle_num_check_manual`（人工盘点记录）、`psi_stall_ai_data`（栏舍AI数据）以及各事务表的关键字段完整性。

### 4. 核心分析指标体系

| 一级指标 | 二级指标 | 计算逻辑 |
|---------|---------|---------|
| **数据完整性** | 关键字段缺失率 | 关键字段为空的记录数 / 总记录数 |
| **数据完整性** | 数据覆盖率 | 有数据记录的牛只数 / 总在栏牛只数 |
| **数据准确性** | 体重异常值比例 | 体重超出合理范围（如<50kg或>1000kg）的记录数 / 总记录数 |
| **数据一致性** | 跨表数据一致性率 | 采购数据与在栏数据匹配的记录数 / 总记录数 |
| **数据及时性** | 平均数据录入延迟 | 实际录入时间 - 业务发生时间的均值 |
| **AI盘点准确性** | AI盘点准确率 | AI盘点数与人工盘点数一致的次数 / 总盘点次数 |
| **AI盘点准确性** | AI盘点误差率 | |AI盘点数 - 人工盘点数| / 人工盘点数的均值 |
| **AI盘点准确性** | AI盘点高误差场景占比 | 误差率>5%的盘点次数 / 总盘点次数 |
| **数据质量趋势** | 月度数据质量评分 | 加权综合多维度质量指标的复合评分 |
| **治理效果** | 数据质量问题修复率 | 已修复的数据质量问题数 / 发现的总问题数 |

### 5. 分析方法与模型

推荐方法包括：**数据剖析（Data Profiling）**用于系统性数据质量评估；**统计过程控制（SPC）**用于数据质量的持续监控；**异常检测算法**（IQR法、Z-Score）用于数据异常值识别；**混淆矩阵分析**用于AI盘点准确率评估；**根因分析（RCA）**用于数据质量问题溯源。

### 6. 分析维度与切片

分析维度包括**数据域维度**（不同业务域的数据质量对比）、**时间维度**（数据质量的改善趋势）、**字段维度**（不同关键字段的质量状况）、**AI盘点场景维度**（不同光照、密度、时间段的盘点准确率）以及**牧场维度**（不同牧场的数据质量对比）。

### 7. 数据产品形态

本专题应呈现为：**数据质量监控看板**（实时显示各域数据质量评分）、**AI盘点准确率分析报告**（月度，含误差分析和改进建议）以及**数据质量问题追踪系统**（记录问题发现、分配、修复的全流程）。

### 8. 实施路径与优先级

**优先级：高**。数据质量是所有分析专题的基础，建议在其他专题启动前，先完成数据质量基线评估。第一阶段（1-2周）完成数据质量基线评估；第二阶段（3-4周）建立数据质量监控体系；第三阶段（5-8周）开展AI盘点准确率专项评估并提出优化建议。核心难点在于建立人工盘点与AI盘点的对比基准，解决思路是设计严谨的抽样复核机制，确保人工复核数据的准确性和代表性。

## 实施建议与总结

牧场业务是一个典型的**物联网 + 供应链金融 + 精细化管理**的复合型业务系统。上述10个分析专题的落地，建议遵循以下三阶段路径：

**第一阶段（基础运营与风控，建议优先启动）** 应优先实施"牛只生长绩效分析"、"供应链金融风控分析"、"饲料成本优化分析"以及"数据质量与AI盘点准确率分析"。这四个专题直接关系到牧场的核心资产安全和盈利能力，数据基础相对较好，且"数据质量分析"是其他所有专题的基础，应最先启动。

**第二阶段（精细化管理与提效）** 推进"疫病健康管理分析"、"栏舍运营效率分析"和"牛只采购与销售分析"。在基础运营稳定的前提下，通过精细化管理进一步降低损耗、提升周转率，实现从"能用"到"好用"的跨越。

**第三阶段（智能化与全面协同）** 落地"AI智能监控预警分析"、"运营任务与工单管理分析"和"牧场综合运营对标分析"。依托物联网和AI技术，实现牧场的全面智能化和跨区域协同管理，形成数据驱动的管理闭环。

在数据仓库建设方面，应严格按照**ODS → DIM → DWD → DWS → ADS**的分层架构，确保数据的一致性、准确性和可用性，为上层分析专题提供坚实的数据底座。特别需要重点建设的基础维度表包括：`dim_customer`（客户维度）、`dim_ranch`（牧场维度，SCD Type 2）、`dim_stall`（栏位维度，SCD Type 2）、`dim_commodity`（商品维度）和`date_dim`（标准日期维度表）。

| 实施阶段 | 涵盖专题 | 建议时间 | 核心目标 |
|---------|---------|---------|---------|
| 第一阶段 | 数据质量、生长绩效、金融风控、饲料成本 | 第1-3个月 | 夯实数据基础，保障核心资产安全 |
