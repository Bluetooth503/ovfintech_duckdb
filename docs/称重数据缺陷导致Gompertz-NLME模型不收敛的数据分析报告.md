# 称重数据缺陷导致 Gompertz + NLME 模型不收敛的数据分析报告

> **报告日期**：2026-04-16
> **数据来源**：`dws_ranch_cattle_adg_agg_i`（ DuckDB 生产/测试环境真实数据）
> **分析对象**：ads_ranch_cattle_gompertz_nlme_pymc_v2 / v3 及 simplified_v2 模型运行结果
> **面向读者**：产品经理、数据分析师、算法工程师、业务决策者

---

## 摘要

本报告基于 DuckDB 中 11 万头肉牛的真实称重记录，系统分析了数据质量缺陷与 Gompertz + NLME（非线性混合效应）模型收敛困难之间的因果关系。

核心发现：
- **76.2%** 的牛只仅有 **1 条** 称重记录，**94.7%** 的牛只称重次数 ≤ 2 次；
- 相邻两次称重的间隔中，**超过 90 天的占比高达 62.2%**，超过 180 天的占 37.7%；
- 对 Gompertz 曲线最关键的"拐点区间"（100-300 天），仅有 **34.5%** 的牛只存在覆盖；
- 在上述数据条件下，PyMC 贝叶斯 NLME 模型即使仅使用 144 头牛，个体拐点日龄 $C$ 的随机效应方差仍**失控到 16,485（标准差 128 天）**，R-hat 指标超标，采样器无法稳定收敛。

**结论**：当前数据稀疏程度已经构成了 NLME 模型不可逾越的"结构性壁垒"。在称重频次和拐点覆盖未显著提升之前，任何对 Gompertz 三参数随机效应模型的复杂度调整（采样策略优化、先验收紧、降维）都难以从根本上解决收敛问题。

## 一、数据来源与样本概况

### 1.1 原始数据规模

| 指标 | 数值 |
|:---|:---|
| 总称重记录数 | **110,244 条** |
| 涉及牛只总数 | **110,244 头** |
| 平均每头牛称重次数 | **1.31 次** |
| 称重次数中位数 | **1 次** |

> 注：原始表为去重后的 `cattle_id` 级别统计，每头牛的平均记录数仅 1.31 条，说明绝大多数牛几乎没有时间序列信息。

### 1.2 有效建模样本

由于 Gompertz 是 3 参数曲线（成年体重 $A$、生长速率 $B$、拐点日龄 $C$），数学上每头牛至少需要 3 次独立观测才能进行最小二乘拟合。若以 NLME 的贝叶斯采样版本（PyMC v2/v3）为例，为保证后验稳定性，通常要求每头牛 ≥ 5 次观测。在此门槛下：

| 观测次数门槛 | 满足条件的牛只数 | 占比 |
|:---|:---|:---|
| ≥ 3 次 | 5,884 头 | 5.3% |
| ≥ 4 次 | 1,578 头 | 1.4% |
| ≥ 5 次 | 414 头 | 0.4% |

**这意味着**：在 11 万头牛中，只有不到 6% 的牛满足"勉强可以画一条曲线"的最低要求；而能支撑贝叶斯 NLME 模型的牛只仅占 **0.4%**。这是模型收敛困难的第一个根源。

## 二、称重数据的三重结构性缺陷

### 2.1 缺陷一：称重频次极低，76% 的牛只称过一次

#### 称重次数分布（真实数据）

| 称重次数 | 牛只数量 | 占比 |
|:---|:---|:---|
| 1 次 | 84,003 | **76.20%** |
| 2 次 | 20,357 | **18.47%** |
| 3 次 | 4,306 | 3.91% |
| 4 次 | 1,164 | 1.06% |
| 5 次 | 333 | 0.30% |
| ≥ 6 次 | 81 | 0.07% |

**分析**：
- **94.7%** 的牛称重次数 ≤ 2 次。对于 Gompertz 这种 S 型曲线，仅有 1-2 个点几乎无法判断曲线的弯曲方向，更遑论估计拐点位置。
- NLME 模型需要"个体内部有足够变异"来估计随机效应。当 76% 的个体只有一个点时，这些个体对随机效应的估计贡献为 0，有效样本被极度压缩。

### 2.2 缺陷二：称重间隔过长，62% 超过 90 天

对于已经满足 ≥ 2 次称重的牛（约 2.6 万头），我们计算了相邻两次称重的日期间隔：

#### 称重间隔分布

| 间隔区间 | 间隔次数 | 占比 |
|:---|:---|:---|
| ≤ 7 天 | 2,615 | 7.64% |
| 8-30 天 | 3,581 | 10.47% |
| 31-60 天 | 3,935 | 11.50% |
| 61-90 天 | 2,793 | 8.16% |
| **91-180 天** | **8,380** | **24.50%** |
| **> 180 天** | **12,904** | **37.72%** |

**关键数字**：
- **间隔 > 90 天的比例：62.2%**（24.5% + 37.7%）
- **间隔 > 180 天的比例：37.7%**

**分析**：
- 牛的育肥关键期通常在 100-300 天（约 3-10 月龄）。如果两次称重间隔超过 180 天，意味着可能直接从"断奶前"跳到"育肥后期"，中间整个猛长期完全缺失。
- Gompertz 拐点 $C$ 的估计极度依赖拐点附近的局部观测。一旦该区间出现长达半年的数据空洞，任何拟合算法都无法确定拐点到底发生在 150 天还是 250 天。

### 2.3 缺陷三：拐点区间覆盖不足，仅 34.5% 的牛只有记录

Gompertz 曲线的数学形态决定了：**如果一头牛在 100-300 天这个区间没有称重记录，它的拐点日龄 $C$ 是不可识别的**。

我们统计了所有牛只在拐点区间的覆盖情况：

| 覆盖情况 | 牛只数量 | 占比 |
|:---|:---|:---|
| 100-300 天**有**称重记录 | 38,033 | **34.50%** |
| 100-300 天**无**称重记录 | 72,211 | **65.50%** |

**也就是说，超过三分之二的牛在生长最关键的"猛长期"没有任何观测。**

#### 主要品种的拐点覆盖情况

| 品种 | 总记录数 | 拐点区间记录数 | 占比 |
|:---|:---|:---|:---|
| 西门塔尔公牛 | 70,865 | 25,555 | **26.46%** |
| 安格斯母牛 | 24,300 | 8,649 | **28.06%** |
| 西门塔尔母牛 | 10,676 | 4,177 | **34.93%** |
| 安格斯公牛 | 980 | 614 | 58.64% |
| 和牛母牛 | 815 | 1,179 | 100.00% |

**分析**：
- 占绝对主流的 **西门塔尔公牛**（7.1 万条记录）仅有 **26.5%** 的记录落在拐点区间。这意味着即使我们只看这个品种，四分之三的数据点要么在断奶前（<100 天），要么在育肥后期（>300 天），无法支撑拐点估计。
- 和牛母牛虽然 100% 的记录都在拐点区间，但样本量仅 815 条，不足以影响整体模型。

### 2.4 牧场级别的称重间隔差异

不同牧场的数据采集规范差异巨大：

| 牧场 | 牛只数 | 称重间隔中位数（天） | 平均间隔（天） |
|:---|:---|:---|:---|
| 现金流测试牧场 | 220 | **417** | 349 |
| 中宁县太阳梁牧场 | 566 | **238** | 230 |
| 甘肃迅驰 | 3,653 | **198** | 214 |
| 八里罕牧场 | 8,036 | **169** | 195 |
| 源里牧场 | 1,756 | **134** | 164 |
| 宁夏海原牧场 | 738 | **97** | 114 |

**分析**：
- 头部牧场（八里罕、甘肃迅驰）的称重间隔中位数高达 169-198 天，意味着平均每半年才称一次重。
- 即使是数据采集最好的 **宁夏海原牧场**，中位数间隔也有 97 天，仍然超过了理想的月度称重标准（30 天）。
- 这种"宽间隔 + 不规则"的数据形态，使得任何基于时间序列的曲线拟合都面临严重的欠定问题。

## 三、数据缺陷如何导致 Gompertz + NLME 模型不收敛

### 3.1 Gompertz 模型的数学依赖：拐点是"命门"

Gompertz 生长曲线的公式为：

$$W(t) = A \cdot \exp(-\exp(-B \cdot (t - C)))$$

其中：
- $A$：渐近体重（天花板）
- $B$：生长速率参数
- $C$：拐点日龄（生长最快的日期）

这个公式的特点是：**$C$ 的微小变化会导致整个曲线形状剧烈变化**。例如：
- 若 $C = 150$ 天，牛在 150 天左右进入猛长期；
- 若 $C = 250$ 天，猛长期被推迟了 100 天，前期曲线会变得非常平缓，后期则非常陡峭。

因此，准确估计 $C$ 需要数据在 $C$ 附近有足够的"锚点"。如果数据在 100-300 天区间是空洞的，$C$ 就会像一个"自由漂浮"的参数，在 100 到 300 之间来回震荡，无法稳定下来。

### 3.2 NLME 的个体级随机效应：从"欠定"到"方差爆炸"

NLME 模型不仅要求估计群体的平均 $C$，还试图估计**每头牛各自的 $C$ 偏差**（随机效应 $u_C$）。这在数据充足时是强大的工具，但在当前数据下成了致命的弱点。

**逻辑链条如下**：

1. **数据稀疏 → 个体 $C$ 不可识别**
   - 对于一头只有 3 条称重记录、且全部落在 <100 天或 >400 天的牛，100-300 天的拐点区间没有任何观测。
   - 数学上，曲线在该区间是"空白"的，$C$ 可以取 150、225 或 280 而几乎不影响拟合残差。

2. **不可识别 → 后验方差失控**
   - 贝叶斯采样器（NUTS）在探索后验分布时，发现 $u_C$ 的分布非常扁平——它可以很大也可以很小，似然函数几乎不惩罚极端值。
   - 为了拟合少数几头有拐点观测的牛，采样器被迫把 $u_C$ 的方差撑得极大。

3. **方差爆炸 → 模型不收敛**
   - 当 $u_C$ 的标准差达到 100 天以上时，部分牛的拐点会被推到出生前（负值）或育肥后期（>400 天），这在生物学上荒谬，但在数学上却"合理解释了数据"。
   - 这种后验几何形状的畸形导致采样器混合困难，R-hat 超标，ESS（有效样本量）暴跌。

### 3.3 真实运行结果：数值层面的崩溃

我们运行了 PyMC v2（三参数随机效应：$A, B, C$）和 PyMC v3（两参数随机效应：$A, C$），结果如下：

#### PyMC v2（144 头牛）

| 诊断指标 | 数值 | 评估标准 | 结论 |
|:---|:---|:---|:---|
| ESS (min) | 325 | > 100 为可接受 | 勉强及格 |
| R-hat (max) | 1.015 | < 1.05 | 正常 |
| Divergences | 0 | = 0 | 正常 |
| **$u_C$ 方差** | **16,485.6** | 生物学合理值 < 900（±30 天） | **严重失控** |
| **$u_C$ 标准差** | **128.4 天** | 合理范围 ±30 天 | **严重失控** |

#### PyMC v3（144 头牛）

| 诊断指标 | 数值 | 评估标准 | 结论 |
|:---|:---|:---|:---|
| ESS (min) | **82** | > 100 | **不足** |
| R-hat (max) | **1.067** | < 1.05 | **超标** |
| Divergences | 0 | = 0 | 正常 |
| **$u_C$ 方差** | **13,160.5** | 合理值 < 900 | **严重失控** |
| **$u_C$ 标准差** | **114.7 天** | 合理范围 ±30 天 | **严重失控** |

**关键解读**：

- **采样器收敛 ≠ 参数合理**：v2 的 R-hat 和 ESS 看起来"数学上收敛了"，但 $u_C$ 方差高达 16,485，意味着模型在技术上跑通了，生物学上却崩盘了。
- **简化模型无效**：v3 砍掉 $B$ 的随机效应后，不但没有改善 $C$ 的稳定性，反而让采样质量更差（ESS 从 325 跌到 82，R-hat 超标）。这证明问题不是"参数太多"，而是**$C$ 本身在当前数据中不可识别**。
- **个体级拐点出现负值**：随机效应结果显示，部分牛的 $u_C$ 偏差低至 -240 天，导致其拐点日龄为负值——"还没出生就开始猛长"，这明确宣告了模型失效。

### 3.4 为什么先验约束也救不了场

为了遏制 $u_C$ 方差失控，我们在 v2/v3 中采用了非常紧的先验约束：
- $u_C$ 的先验设为 `HalfNormal(30)`，意图将个体偏差限制在 ±30 天以内。

**但结果是**：后验标准差仍然突破了 100 天，完全"撑破"了先验。这说明数据中的似然信号极其强烈地要求 $u_C$ 必须有极大的方差，否则就无法解释那些稀疏、偏态的称重记录。当似然与先验冲突到这种程度时，**先验约束失效**。

## 四、模型优化尝试及其局限

在确认数据缺陷是根因之前，我们尝试了多种工程优化：

| 优化策略 | 效果 | 局限 |
|:---|:---|:---|
| **拐点优先采样**（优先选 100-300 天有观测的牛） | 提高了有效样本的拐点覆盖率 | 但总体覆盖率仍太低（仅 34.5%），不足以支撑 144 头牛的个体级 $C$ 估计 |
| **紧约束先验**（HalfNormal(30)） | 未能阻止 $u_C$ 方差爆炸 | 数据似然太强，先验被撑破 |
| **砍掉 $B$ 的随机效应**（v3） | 采样质量反而下降 | 问题不在参数空间大小，而在 $A$-$C$ 联合后验几何形状畸形 |
| **共享 $C$（simplified v2）** | 模型稳定，能输出品种/牧场差异 | 放弃了"个体级拐点预测"，且所有组合的 $C$ 最终都回退到先验 225 天 |

**最终结论**：
- 所有保留"个体级 $C$ 随机效应"的方案（v2、v3）都失败了；
- 唯一稳定的出路是**弱化或移除 $C$ 的个体差异**（共享 $C$），但这意味着 NLME 模型最核心的"个体拐点预测"能力被阉割；
- 真正的瓶颈不是算法，而是**数据在关键生长期（100-300 天）的信息缺失**。

## 五、结论与数据改进建议

### 5.1 核心结论

1. **数据稀疏是结构性问题**：76% 的牛只称过一次，62% 的称重间隔超过 90 天，65.5% 的牛在拐点区间没有观测。这三个数字共同构成了 Gompertz + NLME 模型不可逾越的壁垒。

2. **$C$ 参数不可识别是直接原因**：Gompertz 曲线的拐点估计极度依赖拐点附近的局部数据。当该区间存在长达半年甚至更久的空洞时，个体级 $C$ 成为"自由参数"，后验方差失控到 100 天以上，采样器无法收敛。

3. **算法优化在数据缺陷面前无效**：无论是拐点优先采样、紧约束先验，还是降维（砍掉 $B$ 的随机效应），都没有解决根本问题。所有尝试的 PyMC 模型在技术上要么 $u_C$ 方差爆炸，要么 R-hat 超标。

4. **当前唯一可行方案是简化模型**：将 $C$ 固定为群体共享值（如 225 天），仅保留 $A$ 和 $B$ 的个体差异。这样模型可以稳定运行并输出牧场/品种差异，但代价是失去了个体拐点预测能力。

### 5.2 数据改进建议（按优先级）

#### P0：提升 100-300 天区间的称重频次
- **目标**：将该区间的称重间隔从当前的 **中位数 169 天** 压缩到 **≤ 30 天**；
- **最低要求**：每头牛在 100-300 天期间至少有 **2-3 次** 独立称重；
- **预期效果**：满足此条件后，个体级 $C$ 的可识别性将大幅提升，NLME 模型有望真正收敛。

#### P1：增加每头牛的总称重次数
- **目标**：将"≥ 3 次称重"的牛只比例从 5.3% 提升到 **30% 以上**；
- **最低要求**：至少在断奶前（~60 天）、生长期（~180 天）、育肥后期（~400 天）各有一次称重，形成完整的 S 型轮廓。

#### P2：规范牧场称重节奏
- 针对八里罕（中位数间隔 169 天）、甘肃迅驰（198 天）等头部牧场，建立"月度称重"的 SOP；
- 对现金流测试牧场（417 天）等极端案例，优先排查 IoT 设备或数据同步问题。

#### P3：设计饲料对照实验
- 当前饲料数据内部变化太小，无法做因果推断。建议在同一牧场内选择 2-3 个栏位，做为期 3 个月的配方对照实验，并保证实验期间每周称重，以获得高质量的饲料-生长关系数据。

## 附录：原始数据查询 SQL 汇总

```sql
-- 1. 称重次数分布
SELECT n_obs, COUNT(*) as n_cattle
FROM (
    SELECT cattle_id, COUNT(*) as n_obs
    FROM main.dws_ranch_cattle_adg_agg_i
    WHERE age_days IS NOT NULL AND current_weight > 0
    GROUP BY cattle_id
)
GROUP BY n_obs ORDER BY n_obs;

-- 2. 称重间隔分布
WITH gaps AS (
    SELECT age_days - LAG(age_days) OVER (PARTITION BY cattle_id ORDER BY age_days) as gap
    FROM main.dws_ranch_cattle_adg_agg_i
    WHERE age_days IS NOT NULL AND current_weight > 0
)
SELECT gap_range, COUNT(*) FROM gaps WHERE gap IS NOT NULL GROUP BY gap_range;

-- 3. 拐点区间覆盖
SELECT
    SUM(CASE WHEN has_inflection = 1 THEN 1 ELSE 0 END) as has_inflection_n,
    SUM(CASE WHEN has_inflection = 0 THEN 1 ELSE 0 END) as no_inflection_n
FROM (
    SELECT cattle_id,
           MAX(CASE WHEN age_days >= 100 AND age_days <= 300 THEN 1 ELSE 0 END) as has_inflection
    FROM main.dws_ranch_cattle_adg_agg_i
    WHERE age_days IS NOT NULL AND current_weight > 0
    GROUP BY cattle_id
);
```
