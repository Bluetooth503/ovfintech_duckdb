# 牛只生长 NLME 饲料维度分析实施方案

> 方案：方案 C（饲料结构明细增强）+ 架构 B（两阶段分析）
>
> 目标：在 Gompertz NLME 生长曲线模型中纳入饲料/配方维度，定量分析饲料结构对牛只生长的影响

---

## 一、背景与问题

当前 `ads_ranch_cattle_gompertz_nlme` 模型仅将**品种**和**牧场**作为固定效应，未能充分利用饲料投喂数据。牧场业务中：

- **栏舍级别**已绑定 `recipe_id`（4534 个栏舍对应 235 个配方）
- **牛只级别**有每日饲料投喂明细（`dwd_ranch_cattle_feed_trx_i`），记录了具体饲料商品、投喂量、成本
- **配方级别**有 `ods_psi_recipe`，定义了品种 × 体重区间的标准配方参数

直接将 235 个配方作为固定效应纳入 NLME 会导致维度灾难和 MCMC 收敛失败。因此采用**两阶段分析架构**：

1. 第一阶段：NLME 估计每头牛的个体生长参数（A/B/C）
2. 第二阶段：用饲料结构特征解释个体生长参数的差异

---

## 二、方案架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│  第一阶段：Gompertz NLME 个体参数估计（复用现有模型）                     │
│  ───────────────────────────────────────────────────                    │
│  输入：dws_ranch_cattle_adg_agg_i（增强版）                              │
│  输出：每头牛的 A（成熟体重）、B（生长速率）、C（拐点日龄）估计值         │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│  第二阶段：饲料效应回归分析（独立实现）                                   │
│  ─────────────────────────────────────                                  │
│  输入：个体参数 + 聚合后的饲料结构特征                                    │
│  输出：饲料/配方对生长潜力的定量影响                                      │
└─────────────────────────────────────────────────────────────────────────┘
```

## 三、数据层改造：DWS 表饲料结构增强

### 3.1 改造对象

对 `dws_ranch_cattle_adg_agg_i` 的 `calc_feed_consumption` CTE 进行扩展，在原有的"区间饲料消耗总量"基础上，增加**饲料结构分层聚合**。

### 3.2 饲料分类标准

基于 `ods_psi_commodity` 的 `type` 字段和饲料名称关键词，将饲料划分为：

| 分类 | 判定依据 | 业务含义 |
|------|---------|---------|
| **精料** | `type=2` 且名称含"浓缩料""精料""玉米""豆粕"等 | 高能量蛋白饲料 |
| **粗料** | `type=2` 且名称含"稻草""青贮""秸秆""干草"等 | 纤维类饲料 |
| **添加剂** | `type=2` 且名称含"小苏打""益生菌""舔砖""预混料"等 | 营养添加剂 |
| **药品** | `type=3` | 兽药（单独监控健康成本） |
| **其他** | 未匹配上的饲料商品 | 兜底分类 |

### 3.3 增强后的区间饲料指标

对于每一个"牛只 × 称重区间"，计算：

| 指标名 | 说明 |
|--------|------|
| `period_feed_consumption` | 区间饲料总消耗量（已有） |
| `period_feed_cost` | 区间饲料总成本（已有） |
| `period_concentrate_qty` | 精料消耗量 |
| `period_roughage_qty` | 粗料消耗量 |
| `period_additive_qty` | 添加剂消耗量 |
| `period_medicine_qty` | 药品消耗量 |
| `period_concentrate_ratio` | 精料占比 = 精料量 / 总饲料量 |
| `period_avg_feed_cost_per_kg` | 饲料平均单价 = 总成本 / 总饲料量 |

### 3.4 关联配方信息

通过称重日当天的 `stall_id` 关联 `ods_ranch_stall`，获取该栏舍绑定的：

- `recipe_id`（配方 ID）
- `recipe_name`（配方名称）

同时通过 `sku_id` + `current_weight` 关联 `ods_psi_recipe`，获取该品种在该体重下的：

- `matched_recipe_id`（理论配方 ID）
- `matched_recipe_name`（理论配方名称）
- `recipe_target_fcr`（配方目标料肉比）

> **注意**：`recipe_name` 和 `matched_recipe_name` 仅用于第二阶段的聚类/分组，不直接作为 NLME 第一阶段的输入。

## 四、第一阶段：Gompertz NLME 个体参数估计

### 4.1 输入数据

复用现有 `ads_ranch_cattle_gompertz_nlme.py`，输入增强后的 `dws_ranch_cattle_adg_agg_i`。但**第一阶段 NLME 模型的固定效应保持不变**：

- 品种（`sku_name`）
- 牧场（`ranch_name`）

### 4.2 为什么不把饲料放进第一阶段？

1. **维度控制**：NLME 对固定效应维度极其敏感，每增加一个分类变量都会让 MCMC 参数空间急剧膨胀。
2. **时变协变量的复杂性**：饲料结构是时变的（不同日龄吃不同配方），虽然技术上可以把它作为随时间变化的协变量进入 `X_fixed`，但会大幅增加模型复杂度，收敛风险高。
3. **解耦优势**：先把"生长曲线形状"估计出来，再在第二阶段解释"为什么有的牛曲线更好"，逻辑更清晰，也更稳健。

### 4.3 输出

第一阶段输出每头牛的生长参数估计值，存储为中间表：

| 字段 | 说明 |
|------|------|
| `cattle_id` | 牛只 ID |
| `param_A` | 成熟体重估计值 |
| `param_B` | 生长速率估计值 |
| `param_C` | 拐点日龄估计值 |
| `param_log_A` | `ln(A)`，用于回归（更接近正态分布） |
| `param_log_B` | `ln(B)`，用于回归 |
| `ranch_name` | 牧场（固定效应控制） |
| `sku_name` | 品种（固定效应控制） |
| `n_observations` | 该牛有效称重次数 |
| `convergence_flag` | 个体拟合收敛标记（用于第二阶段清洗） |

## 五、第二阶段：饲料效应回归分析

### 5.1 特征工程：个体级饲料画像

将第一阶段输出的个体参数与增强后的 DWS 表关联，聚合每头牛**全生命周期**或**关键阶段**的饲料特征。

#### 5.1.1 聚合策略

| 策略 | 适用场景 |
|------|---------|
| **全生命周期平均** | 评估牛的总体饲料模式对最终生长潜力的影响 |
| **分阶段聚合** | 按生长阶段（育肥前/育肥后/育成期）分别聚合，分析不同阶段饲料的边际效应 |
| **早期窗口聚合** | 只取入栏后 30 天/60 天的饲料数据，分析早期营养对生长曲线的奠基效应 |

#### 5.1.2 核心特征列表

| 特征名 | 计算方式 | 预期作用 |
|--------|---------|---------|
| `avg_daily_feed_intake` | 总饲料消耗 / 在栏天数 | 日均采食量 |
| `avg_concentrate_ratio` | 精料总量 / 饲料总量 | 精粗比结构 |
| `avg_feed_cost_per_day` | 总饲料成本 / 在栏天数 | 日均饲料成本 |
| `fcr_actual_avg` | 平均实际料肉比 | 饲料转化效率 |
| `recipe_switch_count` | 配方切换次数（按 `recipe_id` 变化计数） | 管理稳定性 |
| `days_on_target_recipe` | 实际投喂配方与理论配方匹配的天数占比 | 配方执行度 |
| `stage_target_fcr_avg` | 各称重日理论配方料肉比的加权平均 | 配方设计预期 |
| `early_concentrate_ratio` | 入栏后 30 天精粗比 | 早期营养强度 |

### 5.2 回归模型选择

因变量为三个个体参数（建议分别建立三个回归模型）：

- `log_A ~ 饲料特征 + 品种 + 牧场`
- `log_B ~ 饲料特征 + 品种 + 牧场`
- `C ~ 饲料特征 + 品种 + 牧场`

#### 模型备选

| 模型 | 特点 | 推荐场景 |
|------|------|---------|
| **OLS + Ridge** | 简单、可解释、带正则化防止过拟合 | 特征数 5~15 个，首选 |
| **随机森林 / LightGBM** | 捕捉非线性交互效应 | 特征数多，存在复杂交互 |
| **贝叶斯线性回归** | 输出置信区间，不确定性量化 | 需要为业务决策提供置信度 |

### 5.3 输出指标

第二阶段输出可用于业务解读的指标表：

| 字段 | 说明 |
|------|------|
| `parameter` | `A`/`B`/`C` |
| `feature_name` | 饲料特征名 |
| `coefficient` | 回归系数 |
| `p_value` / `ci_lower` / `ci_upper` | 显著性检验和置信区间 |
| `interpretation` | 业务解读，如"精料占比每提高 10%，成熟体重 A 增加 X%" |

## 七、落地建议

### 7.1 分阶段实施计划

| 阶段 | 周期 | 任务 | 产出 |
|------|------|------|------|
| **第 1 周** | 数据工程 | 改造 `dws_ranch_cattle_adg_agg_i`，增加饲料结构分层聚合 | 增强版 DWS 表上线 |
| **第 2 周** | 模型改造 | 改造 `ads_ranch_cattle_gompertz_nlme`，增加个体参数落盘 | 第一阶段输出表稳定 |
| **第 3 周** | 特征工程 | 构建 `int_ranch_cattle_feed_profile`，验证特征分布和质量 | 饲料画像表可用 |
| **第 4 周** | 回归分析 | 实现第二阶段回归模型，输出显著性结果和可解释报告 | 最终分析报表上线 |

### 7.2 关键业务问题预设

第二阶段分析应重点回答：

1. **精粗比结构**是否显著影响成熟体重和生长速率？最优区间在哪里？
2. **高成本饲料配方**是否带来了更高的生长回报？（投入产出分析）
3. **频繁切换配方**是否对生长曲线有负面影响？（管理稳定性）
4. **配方执行度**（实际 vs 理论配方的匹配度）对料肉比和生长潜力的影响有多大？
5. **早期营养**（入栏后 30 天饲料结构）对后期生长是否存在长期奠基效应？

## 八、风险与注意事项

### 8.1 数据质量风险

- **饲料分类不准确**：`ods_psi_commodity` 中部分饲料名称不规范，需要维护一套饲料关键词映射字典。
- **投喂明细缺失**：部分历史数据可能存在 `act_quantity` 为 0 或缺失的情况，需要设置合理的数据清洗规则。
- **配方切换时点模糊**：栏舍绑定 `recipe_id` 是"当前状态"，历史时点的配方需要结合 `stall_id` + `date` 推断，可能存在误差。

### 8.2 模型风险

- **传递误差**：第一阶段的个体参数是估计值而非真实值，估计误差会带入第二阶段。建议：
  - 只保留 `n_observations >= 5` 且个体拟合收敛的牛只
  - 对第二阶段系数解读保留适当的保守性
- **共线性**：饲料成本与精料占比通常高度相关，回归中需要检查 VIF，必要时做正则化或降维。
- **遗漏变量偏误**：栏舍管理水平、疫病因素也会影响生长，仅靠饲料特征可能高估其效应。

### 8.3 业务落地风险

- 分析结果指向"某类配方更好"时，需要区分**因果关系**和**相关性**。建议配合 A/B 测试或准实验设计（如换配方前后的同栏舍对比）做验证。

---

## 九、总结

本方案通过**"数据增强 → NLME 分参 → 回归归因"**的三段式架构，在不破坏现有 Gompertz NLME 模型稳定性的前提下，系统性地将饲料/配方维度纳入牛只生长分析。核心设计原则：

- **NLME 只做它擅长的事**：估计非线性生长曲线个体参数
- **回归模型做它擅长的事**：解释高维饲料特征与生长潜力的关系
- **数据工程做桥梁**：通过饲料结构聚合和配方关联，打通投喂明细与生长曲线的血缘

这一架构兼顾了**统计稳健性**和**业务可解释性**，是当前数据条件下的最优落地路径。
