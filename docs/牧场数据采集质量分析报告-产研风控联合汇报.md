## 一、执行摘要

本报告基于 ODS 原始数据层，对牧场项目的牛只数据采集质量进行了全面分析。核心发现如下：

| 问题领域 | 关键指标 | 严重程度 |
|---------|---------|---------|
| **AI 识别** | 区域 AI 识别真实平均约 **63.0%**（排除极端异常值），**43.38%** 的记录 <70%，**23.62%** 触发告警 | 🔴🔴🔴 **致命** |
| **AI 异常** | 存在极端异常值（最大识别率 **283.0%**），大量区域识别率为 0 | 🔴🔴🔴 **致命** |
| **日龄数据** | 原字段仅 **0.06%** 有日龄信息，**通过 birth_date 计算可达 69.53%**（提升 **1,159 倍**） | 🟡🟡 中等 → 可改善 |
| **称重频次** | **88.44%** 的牛仅有 1-2 次称重（173,193/195,827） | 🔴🔴 严重 |
| **称重覆盖** | 称重覆盖率 **82.02%**（134,736/164,279） | 🟡 中等 |
| **日增重异常** | **72.86%** 的牛日增重过低（<0.3 kg/天），**9.06%** 日增重为负，仅 **15.53%** 在正常范围 | 🔴🔴🔴 **致命** |
| **月龄覆盖** | **69.66%** 的称重记录是入栏体重（0-180天），**育肥中期（12-18月）** 覆盖率仅 **2.14%** | 🔴🔴 严重 |

**结论**：牧场项目对牛只的近况数据采集存在**致命缺陷**，特别是：
1. **AI 区域识别系统严重失效**：修正后平均识别率仅 **63.0%**（远低于风控要求的 90%），43.38% 的记录识别率 <70%，存在极端异常值（最高 283.0%）
2. **日增重数据质量严重异常**：**72.86%** 的牛日增重过低（<0.3 kg/天），**9.06%** 日增重为负，仅 **15.53%** 在正常范围（0.3-1.5 kg/天）
3. **日龄数据可通过 birth_date 计算恢复**：虽然原字段仅 0.06% 有数据，但通过 birth_date 计算日龄，覆盖率可达 **69.53%**，提升 **1,159 倍**

这将导致：
- 完全无法通过 AI 监控牛只资产安全（丢失、混入、统计错误）
- **日增重数据不可信**：仅 15.53% 的数据可用于正常的生长分析，大部分数据存在质量问题（日增重过低或为负）
- **已改善**：通过 birth_date 计算日龄，可部分恢复生长曲线分析能力（详见第五章）
- **仍然受限**：69.66% 的称重记录是入栏体重（0-180天），育肥中期（12-18月）覆盖率仅 2.14%

**✅ 重大突破（2026-04-20 更新）**：
通过 `ods_ranch_onstall` 表的 `birth_date`（出生日期）字段，结合称重日期计算日龄：
- **出生日期覆盖率**：69.53%（136,165/195,827）
- **可计算日龄的牛只**：134,021 头
- **称重记录总数**：294,182 条
- **平均日龄**：488.35 天（约 16 个月）
- **⚠️ 极端异常值**：最大日龄 45,981 天（需清洗）

详细分析报告：`docs/月龄测重覆盖率分析报告.md`

**建议**：产研部和风控部**立即**联动，优先解决 AI 识别系统和日龄数据采集问题（包括通过 birth_date 计算日龄的方案），推动牛只体重数据采集流程改革。

## 二、AI 牛只识别率不足

### 2.1 问题现状

AI 区域图像识别系统是牧场数字化监控的核心，通过摄像头识别栏舍内的牛只数量，与系统记录的真实数量进行对比，监控牛只是否存在丢失、混入或统计错误。当前数据表明 AI 识别系统存在严重问题，平均识别率仅为 79.19%，且存在大量极端异常值，无法有效监控牛只资产安全。

### 2.2 数据分析

**查询 SQL：**

```sql
-- 查询1: AI 区域识别整体统计
WITH overall_stats AS (
    -- 原始统计（含异常值）
    SELECT
        COUNT(*) AS total_records,
        ROUND(AVG(ratio), 4) AS avg_rate_all
    FROM ods_psi_region_ai_data
    WHERE value IS NOT NULL
      AND cattle_count IS NOT NULL
      AND cattle_count > 0
),
filtered_stats AS (
    -- 排除 >120% 的极端异常值
    SELECT
        COUNT(*) AS total_records_filtered,
        ROUND(AVG(ratio), 4) AS avg_rate_filtered,
        ROUND(APPROX_QUANTILE(ratio, 0.5), 4) AS median_rate,
        COUNT(CASE WHEN ratio < 0.5 THEN 1 END) AS low_rate_count,
        ROUND(100.0 * COUNT(CASE WHEN ratio < 0.5 THEN 1 END) / COUNT(*), 2) AS low_rate_pct,
        COUNT(CASE WHEN ratio < 0.7 THEN 1 END) AS below_70_count,
        ROUND(100.0 * COUNT(CASE WHEN ratio < 0.7 THEN 1 END) / COUNT(*), 2) AS below_70_pct
    FROM ods_psi_region_ai_data
    WHERE value IS NOT NULL
      AND cattle_count IS NOT NULL
      AND cattle_count > 0
      AND ratio <= 1.2  -- 排除识别率 >120% 的极端异常值
),
abnormal_stats AS (
    -- 统计异常值
    SELECT
        COUNT(*) AS abnormal_count,
        ROUND(100.0 * COUNT(*) / (SELECT COUNT(*) FROM ods_psi_region_ai_data WHERE value IS NOT NULL AND cattle_count IS NOT NULL AND cattle_count > 0), 2) AS abnormal_pct
    FROM ods_psi_region_ai_data
    WHERE value IS NOT NULL
      AND cattle_count IS NOT NULL
      AND cattle_count > 0
      AND ratio > 1.2
)
SELECT
    a.total_records AS 总记录数,
    ROUND(a.avg_rate_all * 100, 2) AS 原始平均识别率_含异常值,
    b.total_records_filtered AS 过滤后记录数,
    ROUND(b.avg_rate_filtered * 100, 2) AS 修正后平均识别率_排除异常值,
    ROUND(b.median_rate * 100, 2) AS 中位数识别率,
    b.low_rate_count AS 低识别率记录_50以下,
    b.low_rate_pct AS 低识别率占比_50以下,
    b.below_70_count AS 低识别率记录_70以下,
    b.below_70_pct AS 低识别率占比_70以下,
    c.abnormal_count AS 异常记录数_120以上,
    c.abnormal_pct AS 异常记录占比
FROM overall_stats a
CROSS JOIN filtered_stats b
CROSS JOIN abnormal_stats c;
```

- **🚨 资产安全无法监控**：修正后平均识别率仅 63.0%，中位数识别率 75%，43.38% 的记录识别率低于 70%，无法准确监控牛只数量，存在资产丢失风险

- **环境因素影响**：光线、遮挡、摄像头角度等环境因素影响识别效果
- **告警处理机制缺失**：大部分牧场合格率（识别率≥90%）低于50%，说明缺乏有效的告警处理流程
- **设备维护不及时**：大量区域长期识别率异常，说明设备维护不到位
- **极端异常值未清洗**：识别率达 283.0% 的异常数据未清洗，影响整体统计
- **数据质量监控缺失**：缺乏统一的AI盘点数据质量监控看板，无法及时发现和处理问题

**短期措施（1-2个月）：**

## 三、牛只体重测量覆盖率不足

### 3.1 问题现状

体重数据是牛只生长监测的核心指标，但当前称重覆盖率极低，大量牛只在栏期间从未称重或仅有极少称重记录。

### 3.2 数据分析

**查询 SQL：**

```sql
-- 查询2: 牛只体重测量覆盖率分析
-- 统计有称重记录的牛只数量 vs 总在栏牛只数量
WITH weighted_cattle AS (
    SELECT DISTINCT livestock_id AS cattle_id
    FROM ods_psi_sample_weight
    WHERE livestock_id IS NOT NULL
      AND CAST(weight AS DOUBLE) > 0
),
onstall_cattle AS (
    -- 获取最新在栏状态的牛只
    SELECT DISTINCT livestock_id AS cattle_id
    FROM ods_ranch_onstall_history
    WHERE livestock_id IS NOT NULL
)
SELECT
    COUNT(DISTINCT o.cattle_id) AS onstall_cattle_count,
    COUNT(DISTINCT w.cattle_id) AS weighted_cattle_count,
    ROUND(100.0 * COUNT(DISTINCT w.cattle_id) / COUNT(DISTINCT o.cattle_id), 2) AS weight_coverage_rate
FROM onstall_cattle o
LEFT JOIN weighted_cattle w ON o.cattle_id = w.cattle_id;
```

**实际数据（基于 ODS 原始数据）：**

| 指标 | 数值 |
|------|------|
| 在栏牛只数量 | 164,279 头 |
| 有称重记录的牛只 | 134,736 头 |
| 称重覆盖率 | **82.02%** |

### 3.3 风险影响

- **生长监测失效**：无法追踪牛只生长曲线，无法及时发现问题
- **出栏决策盲目**：缺乏体重数据支撑，出栏时机判断不准确
- **贷款风险增加**：对于贷款牛只，无法通过体重监测验证资产增值情况
- **饲喂成本失控**：无法根据生长阶段优化饲喂方案，导致饲料浪费

## 四、称重频次严重不足

### 4.1 问题现状

即使有称重记录的牛只，称重频次也严重不足。理想情况下，育肥期牛只应月度称重，但实际情况远低于此标准。

### 4.2 数据分析

**查询 SQL：**

```sql
-- 查询3: 牛只称重频次分布
WITH cattle_weight_count AS (
    SELECT
        livestock_id AS cattle_id,
        COUNT(*) AS weight_count
    FROM ods_psi_sample_weight
    WHERE livestock_id IS NOT NULL
      AND CAST(weight AS DOUBLE) > 0
    GROUP BY livestock_id
),
weight_distribution AS (
    SELECT
        weight_count,
        COUNT(*) AS cattle_count
    FROM cattle_weight_count
    GROUP BY weight_count
    ORDER BY weight_count
)
SELECT
    weight_count,
    cattle_count,
    ROUND(100.0 * cattle_count / SUM(cattle_count) OVER (), 2) AS percentage
FROM weight_distribution
ORDER BY weight_count;
```

**实际数据（基于 ODS 原始数据）：**

| 称重次数 | 牛只数量 | 占比 |
|---------|---------|------|
| 1 次 | 130,477 头 | **66.63%** |
| 2 次 | 42,716 头 | **21.81%** |
| 3 次 | 13,217 头 | **6.75%** |
| ≥ 4 次 | 9,417 头 | **4.81%** |
| **合计（≤2次）** | **173,193 头** | **88.44%** |

### 4.3 风险影响

- **数据不可用**：76% 的牛只仅有 1 次称重，无法计算日增重（ADG）
- **生长曲线缺失**：无法建立牛只生长曲线模型，无法预测出栏时间
- **异常检测失效**：无法通过体重趋势识别生长异常或疾病
- **绩效评估失真**：无法准确评估牧场和养殖员的绩效

## 五、月龄覆盖率严重失衡

### 5.1 问题现状

牛只生长分为**五个关键阶段**：犊牛期（0-6月）、育肥前期（6-12月）、育肥中期（12-18月）、育肥后期（18-24月）、成年期（24月+）。

**✅ 重大突破（2026-04-20 更新）**：
虽然 `weight_day_age` 字段几乎完全缺失（仅 0.06%），但通过 `ods_ranch_onstall` 表的 `birth_date`（出生日期）字段，结合称重日期（`weight_date`）计算日龄，**日龄数据可用性提升 1,159 倍**（从 0.06% → 69.53%）。

**⚠️ 新发现的问题**：
通过 birth_date 计算日龄后发现，**59.32% 的称重记录是入栏体重**（日龄 0-30天），**育肥中期（12-18月，即361-540天）覆盖率严重不足**，无法监控牛只的关键增值阶段。

**📖 生长阶段划分依据**：
基于 Gompertz 生长曲线模型（拐点约225天/7.5月）和行业标准（辽宁省农业农村厅《育肥牛生长阶段划分》）：

### 5.2 数据分析

**方法一：直接使用 weight_day_age 字段（原方法）**

```sql
-- 查询4: 牛只称重月龄覆盖率分析（直接使用 weight_day_age 字段）
WITH cattle_age_coverage AS (
    SELECT
        livestock_id AS cattle_id,
        weight_day_age AS day_age,
        ROW_NUMBER() OVER (PARTITION BY livestock_id ORDER BY weight_date) AS rn
    FROM ods_psi_sample_weight
    WHERE livestock_id IS NOT NULL
      AND CAST(weight AS DOUBLE) > 0
      AND weight_day_age IS NOT NULL
),
age_distribution AS (

            WHEN day_age BETWEEN 100 AND 300 THEN '100-300天(猛长期)'
            WHEN day_age > 300 THEN '300天以上(稳生期)'
        END AS age_range,
        COUNT(DISTINCT cattle_id) AS cattle_count
    FROM cattle_age_coverage
    WHERE rn = 1  -- 每头牛只统计一次

),
total_cattle_count AS (

    WHERE livestock_id IS NOT NULL
      AND CAST(weight AS DOUBLE) > 0

SELECT

    a.cattle_count,
    t.cnt AS total_cattle_count,
    ROUND(100.0 * a.cattle_count / t.cnt, 2) AS coverage_percentage
FROM age_distribution a
CROSS JOIN total_cattle_count t
ORDER BY
    CASE a.age_range
        WHEN '0-99天(慢生期)' THEN 1
        WHEN '100-300天(猛长期)' THEN 2
        WHEN '300天以上(稳生期)' THEN 3
    END;
```

```sql
-- 查询4.5-3: 月龄分布（首次称重，按生长阶段最优分段，含数据质量统计）
-- 说明：
--   - 基于Gompertz生长曲线拐点(约225天/7.5月)和行业标准划分5大生长阶段
--   - 添加中位数、最小值、最大值、标准差以识别异常值
--   - 计算均值与中位数差异，评估数据质量（差异>10%说明存在极端异常值）
-- 参考：段星海(2021)中国肉用西门塔尔牛生长曲线研究(拐点约225天)
WITH cattle_with_birthdate AS (
    SELECT
        code AS cattle_code,
        birth_date
    FROM ods_ranch_onstall
    WHERE code IS NOT NULL
      AND birth_date IS NOT NULL
),
weight_with_calculated_age AS (

        w.code AS cattle_code,
        w.weight_date,
        CAST(w.weight AS DOUBLE) AS weight,
        c.birth_date,
        DATEDIFF('day', c.birth_date, w.weight_date) AS calculated_day_age,

    INNER JOIN cattle_with_birthdate c ON w.code = c.cattle_code

      AND CAST(w.weight AS DOUBLE) > 0
      AND w.weight_date IS NOT NULL
),
monthly_distribution AS (
    SELECT
        CASE
            -- 犊牛期(0-6月): 快速生长期，骨骼和内脏发育
            WHEN calculated_day_age BETWEEN 0 AND 90 THEN '0-90天(0-3月_犊牛前期)'
            WHEN calculated_day_age BETWEEN 91 AND 180 THEN '91-180天(3-6月_犊牛后期)'

            -- 育肥前期(6-12月): 骨架生长期，目标日增重1.1-1.3kg
            WHEN calculated_day_age BETWEEN 181 AND 270 THEN '181-270天(6-9月_育肥前)'
            WHEN calculated_day_age BETWEEN 271 AND 360 THEN '271-360天(9-12月_育肥前)'

            -- 育肥中期(12-18月): 肌肉充实期，关键增值阶段
            WHEN calculated_day_age BETWEEN 361 AND 450 THEN '361-450天(12-15月_育肥中)'
            WHEN calculated_day_age BETWEEN 451 AND 540 THEN '451-540天(15-18月_育肥中)'

            -- 育肥后期(18-24月): 脂肪沉积期，接近出栏标准
            WHEN calculated_day_age BETWEEN 541 AND 660 THEN '541-660天(18-22月_育肥后)'
            WHEN calculated_day_age BETWEEN 661 AND 720 THEN '661-720天(22-24月_育肥后)'

            -- 成年期(24月+): 体成熟，生长缓慢
            WHEN calculated_day_age > 720 THEN '720天以上(24月+_成年期)'

            -- 异常值处理
            WHEN calculated_day_age < 0 THEN '负日龄_异常'
            ELSE '其他'
        END AS age_range,
        COUNT(DISTINCT cattle_code) AS cattle_count,
        ROUND(AVG(weight), 2) AS avg_weight,
        ROUND(MEDIAN(weight), 2) AS median_weight,
        ROUND(MIN(weight), 2) AS min_weight,
        ROUND(MAX(weight), 2) AS max_weight,
        ROUND(STDDEV(weight), 2) AS stddev_weight
    FROM weight_with_calculated_age
    WHERE rn = 1  -- 仅统计首次称重
      AND calculated_day_age >= 0  -- 排除负日龄
    GROUP BY age_range
)
SELECT
    age_range AS 月龄段,
    cattle_count AS 牛只数量,
    ROUND(100.0 * cattle_count / SUM(cattle_count) OVER (), 2) AS 占比_百分比,
    avg_weight AS 平均体重_kg,
    median_weight AS 中位数体重_kg,
    min_weight AS 最小体重_kg,
    max_weight AS 最大体重_kg,
    stddev_weight AS 标准差_kg,
    ROUND((avg_weight - median_weight) / NULLIF(median_weight, 0) * 100, 2) AS 均值中位数差异_百分比,
    CASE
        WHEN age_range LIKE '%犊牛前%' THEN '快速生长期_骨骼发育'
        WHEN age_range LIKE '%犊牛后%' THEN '快速生长期_骨骼发育'
        WHEN age_range LIKE '%育肥前%' THEN '骨架生长期_日增重1.1-1.3kg'
        WHEN age_range LIKE '%育肥中%' THEN '肌肉充实期_关键增值阶段'
        WHEN age_range LIKE '%育肥后%' THEN '脂肪沉积期_接近出栏'
        WHEN age_range LIKE '%成年期%' THEN '体成熟_生长缓慢'
        ELSE '异常'
    END AS 生长阶段说明
FROM monthly_distribution
WHERE cattle_count > 0  -- 仅显示有数据的区间
ORDER BY

-- 参考：段星海(2021)中国肉用西门塔尔牛生长曲线研究(拐点约225天)
WITH cattle_with_birthdate AS (
    SELECT
        code AS cattle_code,
        birth_date

      AND birth_date IS NOT NULL

        w.code AS cattle_code,

    INNER JOIN cattle_with_birthdate c ON w.code = c.cattle_code
    WHERE w.code IS NOT NULL
      AND CAST(w.weight AS DOUBLE) > 0
      AND w.weight_date IS NOT NULL

stage_summary AS (
    SELECT
        CASE
            WHEN calculated_day_age <= 180 THEN '犊牛期(0-6月)'

            WHEN calculated_day_age BETWEEN 361 AND 540 THEN '育肥中期(12-18月)'
            WHEN calculated_day_age BETWEEN 541 AND 720 THEN '育肥后期(18-24月)'
            WHEN calculated_day_age > 720 THEN '成年期(24月+)'
            ELSE '异常'
        END AS growth_stage,
        COUNT(DISTINCT cattle_code) AS cattle_count,
        ROUND(AVG(calculated_day_age), 2) AS avg_day_age
    FROM weight_with_calculated_age
    WHERE rn = 1  -- 仅统计首次称重
      AND calculated_day_age >= 0  -- 排除负日龄
    GROUP BY growth_stage
)
SELECT
    growth_stage AS 生长阶段,
    cattle_count AS 牛只数量,
    ROUND(100.0 * cattle_count / SUM(cattle_count) OVER (), 2) AS 占比_百分比,
    avg_day_age AS 平均日龄_天,
    CASE
        WHEN growth_stage = '犊牛期(0-6月)' THEN '快速生长期_骨骼发育'
        WHEN growth_stage = '育肥前期(6-12月)' THEN '骨架生长期_日增重1.1-1.3kg'

        WHEN growth_stage = '育肥后期(18-24月)' THEN '脂肪沉积期_接近出栏'
        WHEN growth_stage = '成年期(24月+)' THEN '体成熟_生长缓慢'
        ELSE '异常'
    END AS 生长阶段说明
FROM stage_summary
WHERE growth_stage != '异常'
ORDER BY
    CASE growth_stage
        WHEN '犊牛期(0-6月)' THEN 1
        WHEN '育肥前期(6-12月)' THEN 2
        WHEN '育肥中期(12-18月)' THEN 3
        WHEN '育肥后期(18-24月)' THEN 4
        WHEN '成年期(24月+)' THEN 5
        ELSE 99
    END;
```

## 六、入栏体重与日常称重差异分析

### 6.1 问题现状

入栏体重和日常称重是牛只资产估值和生长监测的**两大数据来源**，但两者之间存在显著差异。这些差异可能来自**正常生长**、**测量误差**或**数据记录错误**，需要系统性地分析以识别真正的数据质量问题。

**核心发现**：
- 入栏体重与首次日常称重的**平均间隔长达 208.59 天（约 7 个月）**
- **平均日增重为 1.11 kg/天**（在合理范围内），但**72.86%的牛只日增重过低**（<0.3 kg/天）
- **9.06%的牛只存在负增长**（首次称重 < 入栏体重），明显数据错误
- 短期内（0-7天）日增重高达 **19.73 kg/天**，极端异常

### 6.2 数据分析

**查询 SQL：**

```sql
-- 查询5: 入栏体重与首次日常称重差异分析
-- 说明：
--   - 计算入栏体重与首次日常称重的时间间隔和体重差异
--   - 计算日增重 = (首次称重 - 入栏体重) / 间隔天数
--   - 识别异常数据：日增重为负、过高或过低
WITH install_weights AS (
    SELECT
        code AS cattle_code,
        CAST(weight AS DOUBLE) AS install_weight,
        install_date
    FROM ods_ranch_install
    WHERE code IS NOT NULL
      AND CAST(weight AS DOUBLE) > 0
),
first_daily_weights AS (
    SELECT
        code AS cattle_code,
        MIN(weight_date) AS first_weight_date,
        FIRST_VALUE(CAST(weight AS DOUBLE)) OVER (PARTITION BY code ORDER BY weight_date) AS first_daily_weight
    FROM ods_psi_sample_weight
    WHERE code IS NOT NULL
      AND CAST(weight AS DOUBLE) > 0
    GROUP BY code, weight, weight_date
),
weight_diff_analysis AS (
    SELECT
        i.cattle_code,
        i.install_weight,
        f.first_daily_weight,
        f.first_weight_date,
        i.install_date,
        DATEDIFF('day', i.install_date, f.first_weight_date) AS days_diff,
        -- 体重差异（首次称重 - 入栏体重）
        (f.first_daily_weight - i.install_weight) AS weight_diff,
        -- 日增重
        CASE
            WHEN DATEDIFF('day', i.install_date, f.first_weight_date) > 0 THEN
                (f.first_daily_weight - i.install_weight) / DATEDIFF('day', i.install_date, f.first_weight_date)
            ELSE NULL
        END AS daily_gain
    FROM install_weights i
    INNER JOIN first_daily_weights f ON i.cattle_code = f.cattle_code
    WHERE f.first_weight_date > i.install_date
)
SELECT
    COUNT(*) AS 总牛只数,
    ROUND(AVG(days_diff), 2) AS 平均间隔天数_天,
    ROUND(AVG(install_weight), 2) AS 平均入栏体重_kg,
    ROUND(AVG(first_daily_weight), 2) AS 平均首次称重_kg,
    ROUND(AVG(weight_diff), 2) AS 平均体重差异_kg,
    ROUND(AVG(daily_gain), 2) AS 平均日增重_kg_天,
    -- 体重差异分布
    COUNT(CASE WHEN weight_diff < 0 THEN 1 END) AS 负体重差异_数量,
    ROUND(100.0 * COUNT(CASE WHEN weight_diff < 0 THEN 1 END) / COUNT(*), 2) AS 负体重差异_占比,
    COUNT(CASE WHEN weight_diff > 100 THEN 1 END) AS 体重差异大于100kg_数量,
    ROUND(100.0 * COUNT(CASE WHEN weight_diff > 100 THEN 1 END) / COUNT(*), 2) AS 体重差异大于100kg_占比,
    -- 日增重分布
    COUNT(CASE WHEN daily_gain < 0 THEN 1 END) AS 负日增重_数量,
    ROUND(100.0 * COUNT(CASE WHEN daily_gain < 0 THEN 1 END) / COUNT(*), 2) AS 负日增重_占比,
    COUNT(CASE WHEN daily_gain BETWEEN 0.3 AND 1.5 THEN 1 END) AS 正常日增重_03_15_数量,
    ROUND(100.0 * COUNT(CASE WHEN daily_gain BETWEEN 0.3 AND 1.5 THEN 1 END) / COUNT(*), 2) AS 正常日增重_03_15_占比,
    COUNT(CASE WHEN daily_gain > 1.5 THEN 1 END) AS 日增重过高_大于15_数量,
    ROUND(100.0 * COUNT(CASE WHEN daily_gain > 1.5 THEN 1 END) / COUNT(*), 2) AS 日增重过高_大于15_占比,
    COUNT(CASE WHEN daily_gain < 0.3 THEN 1 END) AS 日增重过低_小于03_数量,
    ROUND(100.0 * COUNT(CASE WHEN daily_gain < 0.3 THEN 1 END) / COUNT(*), 2) AS 日增重过低_小于03_占比
FROM weight_diff_analysis
WHERE days_diff > 0;
```

### 7.1 问题现状

**查询 SQL：**

```sql
-- 查询6: 各牧场数据采集质量对比
WITH ranch_inventory_check AS (
    -- AI 牛只盘点率统计
    SELECT
        tenant_id AS ranch_id,
        COUNT(*) AS ai_check_records,
        COUNT(DISTINCT date) AS ai_check_days,
        ROUND(AVG(ratio), 4) AS avg_ai_check_rate,
        ROUND(APPROX_QUANTILE(ratio, 0.5), 4) AS median_ai_check_rate,
        COUNT(CASE WHEN ratio >= 0.9 THEN 1 END) AS qualified_records,
        ROUND(100.0 * COUNT(CASE WHEN ratio >= 0.9 THEN 1 END) / COUNT(*), 2) AS qualified_rate
    FROM ods_psi_region_ai_data
    WHERE value IS NOT NULL
      AND cattle_count IS NOT NULL
      AND cattle_count > 0
      AND ratio <= 1.2  -- 排除极端异常值
    GROUP BY tenant_id
),
ranch_weight AS (
    SELECT
        tenant_id AS ranch_id,
        COUNT(DISTINCT livestock_id) AS weight_cattle_count,
        COUNT(*) AS total_weight_records
    FROM ods_ranch_install
    WHERE livestock_id IS NOT NULL
    GROUP BY tenant_id
),
ranch_onstall AS (
    SELECT
        tenant_id AS ranch_id,
        COUNT(DISTINCT livestock_id) AS onstall_cattle_count
    FROM ods_ranch_onstall_history
    WHERE livestock_id IS NOT NULL
    GROUP BY tenant_id
)
SELECT
    o.ranch_id,
    o.onstall_cattle_count AS 在栏牛只,
    w.weight_cattle_count AS 称重牛只,
    ROUND(100.0 * w.weight_cattle_count / NULLIF(o.onstall_cattle_count, 0), 2) AS 称重覆盖率,
    ROUND(1.0 * w.total_weight_records / NULLIF(w.weight_cattle_count, 0), 2) AS 平均称重次数_牛,
    a.ai_check_records AS AI盘点记录数,
    a.ai_check_days AS AI盘点天数,
    ROUND(a.median_ai_check_rate * 100, 2) AS AI盘点中位数识别率,
    ROUND(a.qualified_rate, 2) AS AI盘点合格率_90以上
FROM ranch_onstall o
LEFT JOIN ranch_weight w ON o.ranch_id = w.ranch_id
LEFT JOIN ranch_inventory_check a ON o.ranch_id = a.ranch_id
ORDER BY w.weight_cattle_count DESC;
```

**说明**：
- **AI 牛只盘点率**：通过摄像头识别栏舍内的牛只数量，与系统记录对比，监控牛只资产安全
- **AI 盘点中位数识别率**：使用中位数避免极端异常值影响，核心考核指标
- **AI 盘点合格率**：识别率 ≥90% 的记录占比，衡量盘点质量

| **育肥前期** | **6-12月龄，骨架生长期，目标日增重1.1-1.3kg** |
| **育肥中期** | **12-18月龄，肌肉充实期，关键增值阶段（对应Gompertz曲线拐点附近）** |
| **育肥后期** | **18-24月龄，脂肪沉积期，接近出栏标准（600-700kg）** |
| **成年期** | **24月龄以上，体成熟，生长缓慢** |
| **Gompertz拐点** | **肉用西门塔尔牛约为225天（7.5月），是生长速率从加速转为减速的转折点** |
| **计算日龄** | **通过出生日期（birth_date）和称重日期（weight_date）计算的日龄，公式：`DATEDIFF('day', birth_date, weight_date)`。**✅ 已验证**：可将日龄数据可用性从 0.06% 提升至 69.53%，提升 1,159 倍** |
| **出生日期覆盖率** | **有出生日期记录的牛只占总牛只的比例，用于评估通过计算方式获取日龄的可行性。当前为 69.53%（136,165/195,827）** |
| **月龄覆盖率** | **各月龄段的称重记录分布情况。当前问题：59.32% 的称重记录是入栏体重（0-30天），猛长期（100-300天）覆盖率仅 19.01%** |
