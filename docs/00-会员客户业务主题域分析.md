# 会员/客户业务主题域分析报告

## 一、业务概述

基于 `orchid_member` 数据库表结构分析，这是一个**B2B2C供应链金融平台**的会员/客户管理系统，主要服务于农产品供应链场景，支持个人会员、企业机构、金融机构、商户等多类用户，提供认证、授权、支付、签约等全生命周期管理功能。

---

## 二、Kimball 维度建模分析

### 2.1 事实表（Fact Tables）

| 事实表 | 业务含义 | 关键指标 | 粒度 |
|--------|----------|----------|------|
| `operate_log` | 用户操作行为 | 登录/登出/切换机构次数 | 用户 + 时间 |
| `business_opportunity` | 商机交易 | 商机金额、数量 | 商机 |
| `mem_face_verify` | 人脸识别认证 | 识别结果、类型 | 用户 + 业务场景 |
| `mem_credit_apply` | 授权申请 | 授权金额、期限 | 申请单 |
| `customer_tag_assign` | 客户标签分配 | 标签数量 | 客户 + 标签 |

### 2.2 维度表（Dimension Tables）

#### 2.2.1 核心维度

**个人会员维度（`mem_member`）**
- 主键：`id`（会员ID）
- 层级：用户 → 会员 → 关联账户
- 关键属性：姓名、手机、证件、地址、认证状态、标签
- 变化处理：历史记录保存在 `mem_member_history`

**企业机构维度（`mem_company`）**
- 主键：`id`（机构ID）
- 层级：企业 → 员工 → 法人
- 关键属性：企业名称、统一社会信用代码、法人、注册资本、行业、认证状态
- 扩展属性：`mem_company_extra_info`、`mem_company_legal`

**市场维度（`market_participant`）**
- 主键：`market_id`
- 关键属性：市场名称、市场ID

**业务场景维度（`customer_tag_assign`）**
- 主键：`biz_scene_id`、`biz_sub_category_id`
- 关键属性：业务场景、业务子类、资金方、分支机构、监管机构

#### 2.2.2 认证授权维度

**认证状态维度**
- `mem_certified`（个人实名认证）
- `mem_merchant_auth`（商户认证）
- `mem_company` 中的 `is_credit`、`is_core` 字段
- `mem_company_extra_info` 中的工商认证状态

**授权关系维度**
- `mem_company_accredit`（企业员工授权）
- `mem_credit_apply`（授权申请）
- `mem_signature`（安心签授权）
- 关键属性：授权类型、授权期限、授权状态、默认授权人

**CA证书维度**
- `ca_customer_account`（CA证书账户）
- `mem_company` 中的 `ca_status`、`ca_customer_id`
- `mem_member` 中的 `ca_status`、`ca_customer_id`

#### 2.2.3 账户支付维度

**银行账户维度（`mem_bank`）**
- 主键：`id`
- 关键属性：户名、开户行、银行账户、银行卡号、账户类型、可用状态

**发票配置维度（`mem_invoice_config`）**
- 主键：`id`
- 关键属性：航信授权状态、授权账号、授权密钥、授权期限

**关联账户维度（`member_relation_account`）**
- 主键：`id`
- 关键属性：父账户ID、层级关系

#### 2.2.4 流程审批维度

**商机流程维度**
- `business_opportunity`（商机主表）
- `business_opportunity_approval_comment`（商机审批评论）
- `business_entry`（商机准入）
- `business_entry_detail`（商机准入详情）
- `business_entry_workflow`（商机流程）
- `supplier_access_detail`（供应商准入详情）

**客户准入流程维度**
- `customer_entry_workflow`（客户准入流程）
- `corporation_todo`（法人待办）
- `company_attach_change_record`（附件变更记录）

#### 2.2.5 个性化与配置维度

**用户个性化维度**
- `web_column_manager`（个性化字段展示）
- `saved_search`（保存的搜索条件）
- `member_click_view_record`（一键查看权限）

**系统配置维度**
- `mem_online_authority_config`（在线权限配置）
- `mem_app_status`（用户身份状态）

## 三、业务主题域划分

基于表结构分析，会员/客户业务可划分为以下 **7 大主题域**：

### 3.1 会员管理主题域（Member Management）

**核心业务：**
- 个人会员注册、认证、资料管理
- 企业机构注册、认证、资料管理
- 企业员工管理

**核心表：**
- `mem_member`（个人会员主表）
- `mem_member_history`（个人会员历史）
- `mem_member_expand`（个人会员扩展）
- `mem_member_profile_status`（用户资料状态）
- `mem_company`（企业机构主表）
- `mem_company_staff`（企业员工）
- `mem_company_legal`（企业法人信息）
- `mem_company_register`（企业注册记录）
- `mem_company_extra_info`（企业扩展信息）

**关键业务流程：**
1. 用户注册（个人/企业）
2. 实名认证（个人四要素/企业工商认证）
3. 资料完善
4. 账号启用/禁用

### 3.2 认证授权主题域（Authentication & Authorization）

**核心业务：**
- 个人实名认证
- 企业工商认证
- CA证书管理
- 安心签电子签章
- 企业员工授权管理

**核心表：**
- `mem_certified`（个人实名认证）
- `mem_merchant_auth`（商户认证）
- `mem_credit_apply`（授权申请）
- `mem_company_accredit`（企业员工授权）
- `mem_signature`（安心签签名）
- `mem_signature_history`（签名历史）
- `mem_face_verify`（人脸识别认证）
- `ca_customer_account`（CA证书账户）
- `mem_secret`（密钥管理）
- `mem_business_seal`（业务印章）

**关键业务流程：**
1. 个人四要素认证
2. 企业工商信息认证
3. CA证书申请与开通
4. 安心签开通与授权
5. 企业员工授权（经办人/管理员）
6. 人脸识别/活体检测

### 3.3 账户支付主题域（Account & Payment）

**核心业务：**
- 银行账户管理
- 发票配置
- 账户关联
- 支付限额管理

**核心表：**
- `mem_bank`（银行账户）
- `mem_invoice_config`（发票配置）
- `member_relation_account`（关联账户）
- `mem_company` 中的 `cst_acct_no`（九银客户账户）
- `mem_company` 中的 `pay_limits`（支付限额）
- `mem_company` 中的 `pay_sign_status`（九银付签约状态）

**关键业务流程：**
1. 银行账户绑定
2. 账户权限设置
3. 发票开票配置
4. 支付签约与限额设置

### 3.4 商机管理主题域（Business Opportunity）

**核心业务：**
- 商机会员管理
- 商机准入审核
- 商机交易
- 供应商准入评估

**核心表：**
- `business_opportunity`（商机主表）
- `business_opportunity_approval_comment`（商机审批评论）
- `business_entry`（商机准入）
- `business_entry_detail`（商机准入详情）
- `business_entry_workflow`（商机流程）
- `supplier_access_detail`（供应商准入详情）
- `customer_entry_workflow`（客户准入流程）
- `corporation_todo`（法人待办）
- `company_attach_change_record`（附件变更记录）

**关键业务流程：**
1. 商机会员发布
2. 商机准入申请
3. 商机审批流程
4. 供应商准入评估
5. 商机交易执行

### 3.5 客户关系主题域（Customer Relationship）

**核心业务：**
- 客户标签管理
- 客户分类
- 关系人管理
- 地址管理

**核心表：**
- `customer_tag_assign`（客户标签分配）
- `mem_relationship`（关系人）
- `mem_address_info`（地址信息）
- `market_participant`（市场参与者）
- `mem_app_status`（用户身份状态）
- `tmp_regulatory_agency`（监管公司）

**关键业务流程：**
1. 客户标签分配（按业务场景、资金方、分支机构）
2. 客户分类管理
3. 关系人信息维护
4. 地址信息维护
5. 市场参与管理

### 3.6 用户行为主题域（User Behavior）

**核心业务：**
- 用户操作记录
- 登录行为分析
- 权限访问控制
- 个性化设置

**核心表：**
- `operate_log`（操作记录）
- `member_click_view_record`（一键查看权限）
- `web_column_manager`（个性化字段展示）
- `saved_search`（保存的搜索条件）
- `mem_online_authority_config`（在线权限配置）
- `mem_test_account`（测试账户）

**关键业务流程：**
1. 用户登录/登出
2. 机构切换
3. 个性化字段配置
4. 搜索条件保存
5. 一键查看权限管理

### 3.7 系统支持主题域（System Support）

**核心业务：**
- BOA（Bank of America）历史记录
- 系统日志审计
- 数据导入/导出

**核心表：**
- `boa_history`（BOA历史记录）

## 六、核心指标体系

### 6.1 会员管理指标

| 指标名称 | 计算逻辑 | 数据来源 |
|----------|----------|----------|
| 新增会员数 | `COUNT(DISTINCT member_id)` 按日期 | mem_member.create_time |
| 活跃会员数 | `COUNT(DISTINCT member_id)` 有登录记录 | operate_log |
| 实名认证率 | `已认证会员数 / 总会员数` | mem_member.is_credit |
| 企业认证率 | `已认证企业数 / 总企业数` | mem_company.is_credit |
| 会员留存率 | `N日前注册且今日活跃会员数 / N日前注册会员数` | mem_member + operate_log |

### 6.2 认证授权指标

| 指标名称 | 计算逻辑 | 数据来源 |
|----------|----------|----------|
| 认证通过率 | `认证通过数 / 认证申请总数` | mem_credit_apply |
| 授权申请数 | `COUNT(DISTINCT apply_id)` 按日期 | mem_credit_apply |
| CA证书开通数 | `COUNT(DISTINCT ca_customer_id)` 按日期 | ca_customer_account |
| 安心签开通数 | `COUNT(DISTINCT customer_id)` 按日期 | mem_signature |
| 人脸识别成功率 | `识别成功数 / 识别总数` | mem_face_verify |

### 6.3 商机交易指标

| 指标名称 | 计算逻辑 | 数据来源 |
|----------|----------|----------|
| 商机数量 | `COUNT(DISTINCT id)` 按日期 | business_opportunity |
| 商机金额 | `SUM(amount)` 按日期 | business_opportunity |
| 商机转化率 | `成交商机数 / 总商机数` | business_opportunity |
| 商机审批通过率 | `审批通过数 / 审批申请总数` | business_opportunity_approval_comment |
| 供应商准入数 | `COUNT(DISTINCT biz_entry_id)` 按日期 | supplier_access_detail |

### 6.4 账户支付指标

| 指标名称 | 计算逻辑 | 数据来源 |
|----------|----------|----------|
| 银行账户绑定数 | `COUNT(DISTINCT member_id)` 按日期 | mem_bank |
| 支付签约数 | `COUNT(DISTINCT organization_id)` 按日期 | mem_company.pay_sign_status |
| 发票配置数 | `COUNT(DISTINCT member_id)` 按日期 | mem_invoice_config |

认证完成

## 八、数据质量关注点

### 8.1 数据完整性

- **会员信息完整性**：手机号、证件号、地址等关键字段必填
- **企业信息完整性**：统一社会信用代码、法人信息、工商信息必填
- **认证信息完整性**：认证状态、认证时间、认证人必填

### 8.2 数据一致性

- **会员ID一致性**：`mem_member.id`、`mem_company.user_id`、`mem_company_staff.user_id` 需保持一致
- **机构ID一致性**：`mem_company.id`、`mem_company_staff.organization_id`、`mem_signature.organization_id` 需保持一致
- **认证状态一致性**：`mem_member.is_credit`、`mem_company.is_credit`、`mem_certified.is_auth` 需保持一致

### 8.3 数据准确性

- **证件号格式校验**：身份证号、统一社会信用代码格式正确
- **手机号格式校验**：11位手机号格式正确
- **日期格式校验**：有效期、注册时间等日期字段格式正确

### 8.4 数据及时性

- **认证状态更新及时性**：认证状态变更需实时更新
- **授权状态更新及时性**：授权状态变更需实时更新
- **操作日志记录及时性**：用户操作需实时记录

## 九、数据治理建议

### 9.1 主数据管理

- **会员主数据**：以 `mem_member.id` 和 `mem_company.id` 为核心主键，建立会员主数据管理
- **市场主数据**：以 `market_id` 为核心主键，建立市场主数据管理
- **业务场景主数据**：以 `biz_scene_id` 和 `biz_sub_category_id` 为核心主键，建立业务场景主数据管理

### 9.2 数据标准

- **会员ID编码规范**：统一使用 `member_id` 和 `organization_id` 命名
- **日期格式规范**：统一使用 `yyyy-MM-dd HH:mm:ss` 格式
- **状态值规范**：统一使用数字编码（0/1/2/3/...）表示状态

### 9.3 数据血缘

- **ODS → DWD**：原始数据直接同步到明细层
- **DWD → DIM**：从明细层提取维度信息，构建维度表
- **DWD → DWS**：从明细层聚合计算指标，构建汇总表
- **DWS → ADS**：从汇总层进一步聚合计算，构建应用表

### 9.4 数据生命周期

- **操作日志**：保留 6 个月
- **历史记录**：永久保留
- **临时数据**：保留 30 天
- **审批记录**：保留 1 年

## 十、总结

通过对 `orchid_member` 数据库 43 张表的系统分析，会员/客户业务可以划分为 **7 大主题域**：

1. **会员管理主题域**：个人会员、企业机构、企业员工的全生命周期管理
2. **认证授权主题域**：实名认证、工商认证、CA证书、安心签、授权管理
3. **账户支付主题域**：银行账户、发票配置、关联账户、支付签约
4. **商机管理主题域**：商机会员、商机准入、审批流程、供应商评估
5. **客户关系主题域**：客户标签、客户分类、关系人管理、地址管理
6. **用户行为主题域**：操作记录、登录行为、权限控制、个性化设置
7. **系统支持主题域**：BOA历史、日志审计、数据导入导出

基于 Kimball 维度建模思路，构建了完整的**事实表-维度表**体系，明确了数据流向、核心指标体系、关键业务流程和数据治理建议，为后续数据仓库建设提供了清晰的架构蓝图。
