## 📊 系统概述

采价系统（orchid_gather）是一个**多品类、多渠道、多区域的商品价格采集与计算平台**，覆盖农副产品、金属矿产、医药、建材、冻品等 20+ 行业品类。核心能力包括：人工采价、外部渠道采价、均价计算（平均法/部分平均法）、公式计价（稀土/金属等复杂商品）、价格审核与历史追溯。

### 涉及表清单（15 张）

| 表名 | 说明 | 数据特征 |
|------|------|----------|
| `business_rule` | 业务规则配置 | 33 行，定义各业务场景的采价算法和频率 |
| `gather_rule` | 采价规则明细 | 按商品维度配置，关联业务场景 |
| `price_gather` | 采价明细表 | 核心事实表，单条采价记录 |
| `ext_price_gather` | 外部采价数据 | 外部渠道采集的价格数据 |
| `gather_avg_price` | 均价主表 | 日/周/月均价计算结果 |
| `gather_avg_lastest_price` | 最新均价快照 | 每个商品的最新均价状态 |
| `gather_avg_price_history` | 均价历史主表 | 均价时间序列，ID 已达 2200 万+ |
| `gather_avg_price_history_2022` | 2022 年均价历史 | 按年分区 |

| `basic_data_config` | 基础数据配置 | 区域/渠道配置 |
| `sys_config` | 系统配置表 | API 地址、默认参数等 |

### 二、采价数据采集域

人工和外部渠道的价格采集，是系统核心数据来源。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `price_gather` | 采价明细表 | product_id, price, gather_date, channel_id, district_id, business_id, status |
| `ext_price_gather` | 外部采价数据 | prod_name, price, gather_date, channel_name, district_name, business_id, status |

#### 采价渠道

| 渠道类型 | 来源 | 说明 |
|----------|------|------|
| **人工采价** | price_gather | 采集人实地采集（gather_id/gather_name） |
| **外部采价** | ext_price_gather | 从网络等外部渠道批量采集 |

#### 采价审核流程

```
采价录入 → 待审核(3) → 审核通过(1) / 审核拒绝(0)
                    → 正常(-1) → 参与均价计算
                    → 外部采集(2) → 待审核(3) → 通过/拒绝
```

#### 采价数据属性

- **商品维度**: 三级分类（类目→类别→商品）+ 规格属性 + 品牌
- **地域维度**: 区域（district_id/district_name）
- **渠道维度**: 采价渠道（channel_id/channel_name）
- **时间维度**: 采集日期（gather_date）
- **业务维度**: 业务场景（business_id）
- **价格属性**: 单价（price）、单位（unit）、计件单位（weight_unit）
- **批次追踪**: 批次号（batch_num）、条码（bar_code）、编码（code）

### 三、均价计算域

基于原始采价数据计算商品均价，是系统的核心价值产出。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `gather_avg_price` | 均价主表 | prod_id, avg_sort, avg_price, max_price, min_price, period |
| `gather_avg_lastest_price` | 最新均价快照 | prod_id, avg_price, yesterday_price, price_range |
| `gather_avg_price_history` | 均价历史主表 | 同均价主表，按日期追加 |
| `gather_avg_price_history_2022` | 2022 年均价历史 | 按年分区存储 |
| `gather_avg_price_history_2023` | 2023 年均价历史 | 按年分区存储 |

#### 数据规模

- `gather_avg_price` AUTO_INCREMENT = 1.37 亿，数据量极大
- `gather_avg_price_history` AUTO_INCREMENT = 2200 万
- 按 2022/2023 年分区，说明数据量驱动了分区策略

#### 均价计算逻辑

```
原始采价数据(price_gather) → 按商品+日期聚合 → 计算均价/最高价/最低价 → 写入均价主表
                                                                    → 写入历史表
                                                                    → 更新最新快照
```

#### 均价粒度

| 粒度 | period 值 | 说明 |
|------|-----------|------|
| 日均价 | 1 | 每天计算，最常用 |
| 周均价 | 2 | 每周计算 |
| 月均价 | 3 | 每月计算 |

#### 均价输出指标

| 指标 | 字段 | 说明 |
|------|------|------|
| **均价** | avg_price | 当前周期平均价格 |
| **最高价** | max_price | 当前周期最高价格 |
| **最低价** | min_price | 当前周期最低价格 |
| **审核价** | audit_price | 人工审核调整后的价格 |
| **昨日价格** | yesterday_price | 前一周期价格（快照表） |
| **价格变动** | price_range | 涨幅/跌幅金额（快照表） |

#### 商品分类体系

采用**三级分类**结构：

```
一级分类（cate_first） → 二级分类（cate_second） → 三级分类（cate_third）
```

示例：
- 金属 → 稀土 → 镨钕氧化物
- 农产品 → 大蒜 → 大蒜大混级
- 医药 → 西成药 → 阿托伐他汀钙片

### 四、公式计价域

针对稀土、金属废料等需要从原料价格推导成品价格的商品，配置公式化计价规则。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `price_calculate_config` | 计价配置表 | sku_id, sku_name, expression, rule_id |
| `pirce_calculate_ref_sku` | 计价关联 SKU 参数 | config_id, sku_id, price_code, price_code_name, attr_code |
| `price_calculate_condition` | 价格计算条件表 | sku_id, attr_code, min_val, max_val, section_type |

#### 适用商品类型

从数据可见，公式计价主要用于**金属/矿产类**商品：

| 商品名称 | 计价公式引用的原料 |
|----------|-------------------|
| 中亿富铕矿 | 镨钕氧化物价格、氧化铽价格、氧化镝价格、氧化镥价格 |
| 废磁钢 | 钕铁硼废料镨钕价格、铽价格、镝价格、钆价格 |
| 废钨棒材 | 钴粉价格 |
| 废钨磨削料 | 废钨磨削料价格 |
| 废钨丝 | 钴粉价格 |
| 废钨合金刀片 | 钴粉价格 |

#### 公式计价逻辑

```
原料价格（从均价表获取）→ 代入公式表达式（expression）→ 计算成品价格
                    → 可加区间条件（price_calculate_condition）
```

#### 条件区间类型

| section_type | 含义 | 示例 |
|--------------|------|------|
| 0 | 开区间 | (min, max) |
| 1 | 闭区间 | [min, max] |
| 2 | 左开右闭 | (min, max] |
| 3 | 左闭右开 | [min, max) |

## 🎯 核心业务流程

### 采价全流程闭环

```
业务规则配置 → 采价规则配置 → 价格采集（人工/外部） → 审核校验 → 均价计算 → 历史存储 → 价格输出
                                          ↗                        ↘
                               公式计价（稀土/金属）← 原料均价 ←----
```

### 采价录入流程

```
选择业务场景 → 选择商品（三级分类） → 录入价格 → 提交审核 → 审核通过/拒绝
                                               → 选择渠道/区域
                                               → 填写规格/品牌/单位
```

### 均价计算流程

```
筛选有效采价数据 → 按商品+日期分组 → 计算均价/最高/最低 → 写入均价表 → 更新快照 → 记录历史
     ↓                                                                    ↓
 排除过期数据（expire_period）                                      计算涨跌幅（vs 昨日）
```

### 公式计价流程

```
配置 SKU 计价公式 → 关联原料 SKU 价格变量 → 获取原料最新均价 → 代入公式计算 → 输出成品价格
                                        → 校验属性区间条件
```

## 🔍 业务架构特点

### 1. 多行业多品类

- **覆盖 20+ 行业品类**，从农副产品（大蒜/辣椒/水果）到工业原料（金属/钢铁/石油）
- **三级分类体系**，支持精细化的商品管理
- **灵活的业务场景配置**，每个品类独立规则

### 2. 多渠道采集

- **人工采集**: 实地采价，含采集人信息
- **外部采集**: 网络/第三方渠道，批量导入
- **渠道差异化**: 不同渠道有不同默认值（网络→全国）

### 3. 双重计价模式

- **简单均价**: 平均法/部分平均法，适用于标准化商品（大蒜/玉米/药品等）
- **公式计价**: 基于原料价格的数学表达式，适用于复杂商品（稀土矿/废钨/钴粉等）

### 4. 数据量级

- `gather_avg_price` 主表 ID 已达 **1.37 亿**，日均数据量巨大
- `gather_avg_price_history` 达 **2200 万+**，按年分区管理
- 历史数据按年分表（2022/2023），体现数据增长驱动的设计演进

### 5. 审核质控

- **双重审核**: 采价数据审核 + 均价审核（audit_price）
- **操作日志**: 全程记录审核操作，可追溯
- **失效周期**: 自动过期机制，保障数据新鲜度

## 🏭 生产环境数据管道

> 基于海豚调度工作流（2026-04-21 导出）梳理

### 生产工作流概览

采价数据管道嵌入在 WMS 工作流中，由 2 个子工作流组成：

| 工作流名称 | 说明 | 调度方式 |
|-----------|------|---------|
| `ods_dwd_gather_avg_price_i_d` | 价格采集数据增量同步 | 每日调度 |
| `ods_dwd_gather_avg_price_历史数据计算` | 历史数据回填 | 手动触发 |

**数据源**: `orchid_gather.gather_avg_lastest_price`（MySQL 业务库，过滤测试数据）
**目标库**: `wms_prd`（Doris 数仓）

### 生产数据流向图

```
┌──────────────────────────────────────────────────────────────────────────────┐
│  MySQL 业务库 (orchid_gather)                                                │
│  └── gather_avg_lastest_price  (过滤: business_id != 'ceshi'                │
│                                  AND prod_name NOT LIKE '%测试%')            │
│         │ DataX                                                              │
│         ▼                                                                    │
```

## 📝 总结

采价系统是一个**多行业、多渠道、多模式的商品价格采集与计算平台**：

1. **行业覆盖广**: 20+ 行业品类，从农副产品到工业原料
2. **采集渠道多**: 人工实地采集 + 外部网络批量采集
3. **计价模式灵活**: 简单均价法 + 公式计价法（适配标准化与非标商品）
4. **数据体量大**: 均价主表 1.37 亿行，历史表按年分区
5. **质控体系完善**: 双重审核、操作日志、失效周期

### 生产环境现状

生产已实现 **4 张表**（ODS/DWD/DWS/ADS），形成完整的增量管道，核心功能为均价趋势计算（周同比/年同比/僵化检测）。采价数据是**库存货值计算和风控质押率**的关键上游。

### 优化建议

1. **ODS→DWD 为 1:1 直传**: DWD 层未做任何清洗/标准化，可考虑合并或增加数据质量校验
2. **采价域挂在 WMS 库下**: 采价是独立业务域，当前表名前缀为 `wms_prd`，dbt 重构时建议独立为 `gather` 域
3. **价格僵化检测只输出 0/1**: `price_unchanged_in_last_*` 字段仅标识是否变动，未计算具体波幅，建议增加波幅率指标
4. **缺乏采价覆盖率指标**: 生产中未体现采价覆盖率、采集完成率等数据质量指标
5. **缺乏品类维度表**: 商品分类信息通过 JOIN SKU 维度获取，建议在采价域内建立独立的品类维度表
6. **大量业务表未入仓**: `price_gather`（采价明细）、`ext_price_gather`（外部采价）、`gather_rule`（采价规则）、`price_calculate_config`（公式计价）等 10 张表生产中尚未同步入仓

建议在数据仓库建设时，重点突出：
- **价格趋势**分析（日/周/月价格走势、涨跌幅）
- **采价覆盖**分析（品类覆盖率、区域覆盖率、渠道覆盖率）
- **数据质量**监控（审核通过率、采集及时率、数据新鲜度）
- **品类对比**分析（同品类跨区域价差、跨品类价格相关性）
- **公式计价**追溯（原料价格变动→成品价格变动传导分析）
