## 📊 会员/客户业务主题域划分

### 一、会员/客户基础域

管理个人会员、企业会员及其员工的基础信息，是整个系统的核心主数据域。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `mem_member` | 个人会员表 | user_id, name, mobile, idNo, idType, is_credit |
| `mem_company` | 企业会员表 | comp_name, short_name, idNo, idType, capital, industry, type |
| `mem_company_staff` | 企业员工表 | user_id, organization_id, staff_name, type, is_operator |
| `mem_company_legal` | 法人信息表 | org_id, legal_mobile, legal_card_expire_date |
| `mem_relationship` | 关系人表 | member_id, relation_member_id, relation |
| `mem_member_expand` | 会员扩展表 | user_id, member_id, contact_phone, bank_name, acct_no |

#### 业务价值

- **会员管理**: 维护个人会员和企业会员的完整基础信息
- **组织架构**: 管理企业-员工的组织关系和层级结构
- **关系网络**: 维护会员之间的关系网络（配偶、亲属、担保等）
- **实名认证**: 支持个人和企业的实名认证状态管理

#### 数据特点

- **高基数**: mem_member 表 AUTO_INCREMENT=531043，数据量大
- **快速增长**: mem_company_staff 表 AUTO_INCREMENT=16481，持续增长
- **多层结构**: 个人 → 企业 → 员工的三层组织架构

---

### 二、认证/鉴权域

提供多维度的身份认证和授权服务，确保业务安全性和合规性。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `mem_certified` | 实名认证表 | member_id, id_no, is_auth, mobile, signature |
| `mem_face_verify` | 人脸识别表 | member_id, organization_id, type, type_code, result |
| `mem_signature` | 电子签名表 | member_id, organization_id, customer_id, seal_type, is_certi |
| `mem_company_accredit` | 企业授权表 | organization_id, member_id, auth_start, auth_end, type, status |
| `mem_credit_apply` | 信用申请表 | organization_id, member_id, financial_product_id, auth_status |
| `ca_customer_account` | CA证书表 | ca_user_id, identity_no, certificate_type, open_status |
| `mem_merchant_auth` | 商户认证表 | member_id, member_type, auth_status, tax_qualification |

#### 认证类型

**身份认证**
- 实名认证（四要素验证）
- 人脸识别/活体检测
- 电子签名

**企业认证**
- 工商认证（统一社会信用代码）
- 企业四要素认证
- 法人认证

**专业认证**
- CA证书认证（安心签、北京CA、法大大CA）
- 商户认证（税务资质）
- 信用申请授权

#### 业务流程

```
用户注册 → 实名认证 → 人脸识别 → 开通电子签名 → 企业授权 → 业务办理
```

#### 关键指标

- 认证通过率
- CA证书开通率
- 电子签名覆盖率
- 商户认证通过率

### 三、商机/业务准入域

管理商机的全生命周期，从商机发现、准入审批到落地实施。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `business_opportunity` | 商机表 | customer_name, biz_scene, fund_org_id, status, land_plan_date |
| `business_entry` | 业务准入表 | member_id, market_id, access_status, start_time, pass_time |
| `business_entry_detail` | 准入详情表 | biz_entry_id, main_business, total_assets, cooperation_scale |
| `supplier_access_detail` | 供应商准入详情 | biz_entry_id, main_business, operation_condition, access_valid_date |
| `business_opportunity_approval_comment` | 商机审批评论 | business_opportunity_id, serial_no, approve_comment |
| `boa_history` | 商机审批历史 | business_opportunity_id, approval_status, approval_no |
| `business_entry_workflow` | 准入工作流 | id, workflow_info |
| `customer_entry_workflow` | 客户准入工作流 | org_id, serial_no |

#### 业务流程

```
商机发现 → 商机提报 → 准入审批 → 风险评估 → 准入通过 → 业务落地
```

#### 商机状态

- **商机状态**: status（待处理、审批中、已通过、已拒绝、已撤回）
- **准入状态**: access_status（待准入、审批中、审批通过、审批拒绝、审批撤回）

#### 关键字段

**业务信息**
- 业务场景（biz_scene）
- 业务子类（biz_sub_category）
- 资金机构（fund_org_id）
- 分支机构（sub_org_id）

**规模信息**
- 总业务规模（biz_scale）
- 可合作规模（cooperate_scale）
- 月收入评估（monthly_income）
- 收费标准（charge_stand）

**风控信息**
- 公司担保（corporate_guarantee）
- 管控方案（manage_program）
- 上下游信息（up_down_comp）
- 仓库信息（warehouse_info）

## 🎯 核心业务闭环

### 会员注册闭环

```
用户注册 → 实名认证 → 人脸识别 → 完善信息 → 开通服务
```

### 企业入驻闭环

```
企业注册 → 工商认证 → 法人认证 → 添加员工 → 授权开通 → 商务合作
```

### 商机业务闭环

```
商机发现 → 商机提报 → 准入审批 → 风险评估 → 准入通过 → 业务落地 → 持续运营
```

### 认证授权闭环

```
身份认证 → 人脸识别 → CA证书 → 电子签名 → 企业授权 → 业务办理
```

## 🔍 业务架构特点

### 1. 多租户架构

- **市场隔离**（market_id）：支持多个市场的数据隔离
- **机构层级**：个人会员 → 企业会员 → 企业员工的三层架构
- **权限分级**：经办人、管理员、法人的不同权限级别

### 2. 强认证体系

- **多重认证**：实名认证 + 人脸识别 + CA证书 + 电子签名
- **企业认证**：工商认证 + 企业四要素 + 法人认证
- **专业认证**：商户认证（税务资质）

### 3. 全流程风控

- **准入管理**：商机准入、供应商准入的严格审批
- **风险评估**：财务状况、经营状况、信用记录的全面评估
- **变更监控**：证件变更记录、操作日志的完整追踪

### 4. 灵活标签体系

- **多维度标签**：业务场景、资金方、监管公司的多维度标签
- **动态分配**：客户标签的动态分配和调整
- **精准营销**：基于标签的精准客户管理

### 5. 丰富的客户视图

- **基础信息**：个人/企业的完整基础信息
- **关系网络**：关系人、关联账户的关系图谱
- **行为记录**：操作日志、搜索记录、查看记录
- **业务状态**：商机状态、认证状态、授权状态

## 📝 总结

会员/客户业务是一个典型的**多租户 + 强认证 + 商机管理**的综合型客户管理系统：

1. **多租户架构**: 支持多市场、多机构的数据隔离和权限管理
2. **强认证体系**: 实名认证 + 人脸识别 + CA证书 + 电子签名的多重认证
3. **商机全流程**: 从商机发现、准入审批到业务落地的完整闭环
4. **精细化运营**: 客户标签、行为分析、风险评估的全方位管理
5. **合规安全**: 证件变更记录、操作日志、审批历史的完整审计

建议在数据仓库建设时，重点突出：
- **会员增长**分析（注册趋势、认证转化、活跃留存）
- **商机运营**效率（商机转化率、审批时长、准入通过率）
- **企业风险**监控（认证状态、变更记录、经营状况）
- **行为洞察**分析（登录行为、操作偏好、业务场景）
