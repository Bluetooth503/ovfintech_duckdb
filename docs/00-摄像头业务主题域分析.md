## 二、业务背景

该系统是**面向猪场（养殖场）的物联网摄像头监控平台**，核心能力是：

1. 在猪场大门等关键位置部署摄像头设备
2. 通过AI/图像识别自动抓拍进出车辆
3. 识别车牌号、车辆颜色、车辆类型（卡车/小汽车/越野车/四驱车等）
4. 记录车辆进出场事件（type: 0=进场, 1=出场）
5. 事件可人工标注（remark: 饲料车、接货、未出场等）
6. 事件有发布状态管理（status: 0=未发布, 1=已发布, 2=取消）

**典型业务场景**：供应链金融风控——通过摄像头监控猪场车辆进出，掌握饲料运输、生猪出栏等关键物流信息，辅助贷后管理和风险预警。

## 三、主题域划分

### 1. 客户/猪场主数据域

管理客户（养殖企业）及其下辖猪场的基础信息，是整个系统的组织骨架。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `customer` | 客户（养殖企业） | customer_name, customer_code, member_ids |
| `piggery` | 猪场 | piggery_name, piggery_code, loan_amount |

#### 数据特征

- **客户**：正邦、双胞胎等大型养殖企业，通过 `member_ids` 关联多个业务成员
- **猪场**：每个猪场归属于一个客户，携带 `loan_amount`（贷款金额）字段，与金融业务直接关联
- **层级关系**：客户 → 猪场（一对多）

#### 业务价值

- 租户隔离：按客户维度管理数据和设备
- 风控锚点：猪场是贷款风控的基本单元，`loan_amount` 直接关联金融场景
- 组织架构：为客户提供多猪场统一管理视图

---

### 2. 设备管理域

管理部署在猪场现场的IoT设备（摄像头、地磅），以及摄像头的抓拍策略配置。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `piggery_device` | 猪场设备 | piggery_id, type, name, no |
| `capture_camera` | 抓拍摄像头配置 | device_id, name, no, channel, cap_strategy |

#### 数据特征

- **设备类型**：type=0 摄像头，type=1 地磅（当前抽样仅见摄像头）
- **设备命名**：`{客户}{猪场}{位置}({设备编号})`，如"正邦大塘猪场二级隔离区大门(D66172916)"
- **抓拍策略**（cap_strategy）：JSON格式，描述按时间间隔的抓拍规则
  ```json
  [{"time":0,"num":1}, {"time":60,"num":1}, {"time":90,"num":1}, {"time":120,"num":1}, {"time":130,"num":1}]
  ```
  含义：在第0秒、60秒、90秒、120秒、130秒各抓拍1张
- **多通道**：一个物理设备可通过不同 channel 配置多个抓拍点（如"大门背后"、"大门内"）

#### 业务价值

- 设备全生命周期管理（启用/禁用/删除）
- 灵活配置抓拍策略，适配不同监控场景
- 设备与猪场绑定，实现精准定位

---

### 3. 车辆事件监控域（核心业务域）

通过摄像头自动识别和记录车辆进出猪场的实时事件，是整个系统的核心业务数据。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `device_event` | 设备事件（车辆进出记录） | serial_no, content, device_id, car_no, type, status, remark |

#### 事件数据结构

**事件内容格式**：`车辆{车牌号},{颜色},{车型}`

| 字段 | 说明 | 示例值 |
|------|------|--------|
| serial_no | 流水号（时间+车牌） | 20200928151239FA9517 |
| content | 事件内容 | 车辆黄赣FA9517,黑色,卡车 |
| car_no | 车牌号 | 黄赣FA9517、无车牌 |
| type | 进出类型 | 0=进场, 1=出场 |
| status | 发布状态 | 0=未发布, 1=已发布, 2=取消 |
| remark | 人工备注 | 饲料车、接货、未出场、接收货物 |

#### 车辆识别维度

从 `content` 字段可提取三个维度：

| 维度 | 提取值 | 说明 |
|------|--------|------|
| 车牌 | 有牌/无牌 | 是否识别到车牌号 |
| 颜色 | 黑/白/蓝/黄/其他 | 车辆外观颜色 |
| 车型 | 卡车/小汽车/越野车/四驱车/公交/面包车 | 车辆类型分类 |

#### 事件状态流转

```
未发布(0) → 已发布(1)     确认事件有效
未发布(0) → 取消(2)       标记为误识别或无效事件
```

#### 业务价值

- **车辆进出监控**：实时掌握猪场物流动态
- **车牌识别追溯**：可追踪特定车辆的进出历史
- **物流信息提取**：从备注中可获取"饲料车"、"接货"等物流语义
- **风控依据**：结合贷款金额，监控异常物流（如频繁运出可能涉及资产转移）

## 五、核心业务闭环

### 车辆监控闭环

```
摄像头部署 → 抓拍策略配置 → 自动识别 → 事件记录 → 人工审核(发布/取消) → 数据分析
```

### 风控闭环

```
贷款发放 → 猪场设备部署 → 车辆进出监控 → 异常行为发现 → 风险预警
```

## 七、数据特点与业务洞察

### 1. 物联网数据特征

- **高频事件**：device_event 自增到 4252，且抽样数据中单日事件密集（一天数十条），说明摄像头自动识别频率很高
- **半结构化数据**：cap_strategy 为 JSON，content 含结构化文本（车牌/颜色/车型），需要解析
- **低识别率**：大量"无车牌"记录，说明车牌识别准确率有提升空间

### 2. 供应链金融场景

- 猪场直接关联 `loan_amount`（贷款金额），说明摄像头监控服务于贷后风控
- 通过车辆进出监控，可推断饲料运输、生猪出栏等关键经营信息
- remark 中的"接货"、"未出场"、"饲料车"等标注，体现人工审核的业务语义

### 3. 猪场业务（区别于牧场）

- 该系统服务于**养猪场**（piggery），而非此前的牧场（ranch/牛只）
- 客户为大型养殖集团（正邦、双胞胎），业务模式为集团化多猪场管理
- 监控对象是**车辆物流**而非动物个体，与牧场域的牛只追踪不同

### 4. 数据质量观察

- `content` 字段中有格式不一致的情况（如"黄色,，面包车"存在多余逗号）
- 无车牌记录占比较高，可能影响追踪准确性
- 部分事件缺少 remark，人工审核覆盖率需关注

## 九、总结

摄像头（IoT Camera）主题域是一个**物联网 + 供应链金融风控**的垂直系统：

1. **核心功能**：通过部署在猪场大门的摄像头，自动识别和记录车辆进出事件
2. **业务目标**：为供应链金融提供贷后风控数据——监控猪场物流动态，掌握饲料输入和生猪输出情况
3. **数据流向**：设备管理 → 自动抓拍 → 车辆识别 → 事件记录 → 人工审核 → 数据分析
4. **与牧场域的关系**：同属养殖行业，但监控对象不同（车辆 vs 牛只），是平行的业务主题域

建议在数据仓库建设时，重点关注：
- **车辆识别准确率**：车牌识别率、无牌车处理策略
- **物流监控分析**：饲料运输频次、生猪出栏节奏
- **风控预警体系**：异常进出、夜间活动、未审核事件告警
- **跨域联动**：与牧场域的客户维度打通，构建统一的养殖行业风控视图
