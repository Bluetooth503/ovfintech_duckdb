## 📊 牧场业务主题域划分

### 一、基础主数据域

管理业务核心主数据，为其他域提供基础支撑。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `psi_customer` | 客户表 | customer_type, customer_name, status |
| `ranch` | 牧场表 | name, name_abbr, address, owner, code |
| `ranch_stall` | 牛舍/栏位表 | name, recipe_id, system_cattle_num, weight |
| `psi_commodity` | 商品表(饲料/药品) | code, name, type, brand, spec |
| `psi_warehouse` | 仓库表 | code, name, owner_id, manager |
| `psi_driver` | 驾驶员表 | name, group_name, enable |
| `psi_region` | 区域表 | - |

#### 业务价值

- **客户管理**: 区分供应商、借款人等不同客户类型
- **牧场管理**: 维护牧场基本信息、位置、负责人
- **栏位管理**: 细化到每个牛舍/栏位的配置和配方
- **商品管理**: 统一管理饲料、药品等物资主数据

---

### 二、资产/牛只域

牧场核心资产，贯穿整个业务流程。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `ranch_onstall` | 在栏牛只表 | code, weight, price, install_date, is_loan, latest_weight |
| `psi_cattle_purchase` | 牛只采购单 | code, customer_id, quantity, purchase_cost |
| `psi_cattle_sell` | 牛只销售单 | code, customer_id, quantity, sell_cost |
| `psi_cattle_return` | 牛只退回单 | code, return_weight, return_price, reason |
| `psi_cattle_ai_score_result` | 牛只AI评分 | weight, score, hair, muscle, out_stall_weight |
| `psi_stall_ai_data` | 栏舍AI数据 | stall_id, cattle_count, value, act_count |
| `psi_cattle_price` | 牛只价格配置 | commodity_id, price, start_weight, end_weight |
| `psi_cattle_grow_config` | 牛只生长配置 | start_weight, end_weight, plan_weight_add, days |
| `psi_valuable_cattle_evaluation` | 精品牛评估 | - |

#### 业务流程

```
牛只采购 → 入栏安装 → 在栏管理(生长监控/AI评分) → 出栏销售 → 疑难退回
```

#### 关键指标

- 入栏重量/价格、当前体重、预估出栏体重
- 生长速度、料肉比、膘情评分
- 质押状态、贷款关联

---

### 三、贷款/金融域

基于牛只资产的供应链金融服务。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `psi_loan_plan` | 贷款计划表 | code, loan_amount, cattle_num, contract_no, status |
| `psi_loan_plan_cattle_detail` | 贷款牛只明细 | - |
| `psi_loan_repay` | 还款计划表 | amount, type, customer_id, contract_no |
| `psi_loan_repay_cattle_detail` | 还款牛只明细 | - |
| `psi_loan_repay_debt_detail` | 还款债务明细 | - |
| `ranch_loan` | 牧场贷款余额 | ranch_id, loan_balance |
| `psi_loan_config` | 贷款配置 | - |
| `psi_debt` | 债务表 | - |

#### 业务模式

- **融资模式**: 以牛只为质押物的活体抵押贷款
- **放款流程**: 牛只采购入栏 → 质押 → 放款 → 按重量增信还款
- **还款方式**:
  - 按重量增信还款 (weight_add_amount)
  - 出栏还款 (in_stall_amount)
  - 其他方式 (other1_amount, other2_amount)

#### 关键指标

- 贷款余额、用信率
- 质押率 (货值/贷款余额)
- 贷款到期天数

## 🎯 核心业务闭环

### 资产闭环

```
采购入栏 → 在栏管理 → 出栏销售 → 疑难退回
```

### 资金闭环

```
质押放款 → 生长监控 → 按重增信还款 → 出栏结清
```

### 成本闭环

```
采购入库 → 饲喂消耗 → 成本核算 → 利润分析
```

### 监控闭环

```
AI监控 → 预警发现 → 工单处理 → 巡检验证
```

## 📈 关键业务指标

| 类别 | 指标 | 说明 |
|------|------|------|
| **资产指标** | 在栏数量、入栏重量、出栏重量、增重速度 | 牛只资产管理 |
| **资金指标** | 贷款余额、用信率、质押率、还款进度 | 供应链金融 |
| **成本指标** | 饲料成本、料肉比、日投喂量、服务费 | 成本控制 |
| **健康指标** | 病牛率、治愈率、免疫覆盖率、死亡率 | 健康管理 |
| **运营指标** | 巡检评分、预警处理及时率、工单完成率 | 运营效率 |

## 🔍 业务架构特点

### 1. 物联网深度融合

- 视频监控全覆盖
- AI智能识别牛只状态
- 自动预警机制
- 设备离线监控

### 2. 供应链金融模式

- 活体资产质押
- 按重增信还款
- 实时资产监控
- 风险预警体系

### 3. 精细化管理

- 配方级饲料管理
- 栏舍级成本核算
- 牛只级健康档案
- 个体级生长追踪

### 4. 全流程数字化

- 从采购到销售全程在线
- 从放款到还款闭环管理
- 从监控到工单自动流转
- 从计划到执行可追溯

## 📝 总结

牧场业务是一个典型的**物联网 + 供应链金融 + 精细化管理**的复合型业务系统：

1. **资产数字化**: 每头牛都有唯一编码，全生命周期跟踪
2. **风控智能化**: AI监控 + 人工巡检 + 预警体系
3. **金融场景化**: 基于活体资产的供应链金融产品
4. **运营精细化**: 栏舍级管理、配方级饲喂、个体级健康

建议在数据仓库建设时，重点突出：
- **资产流动性**分析(入栏、在栏、出栏周转)
- **金融风险**监控(质押率、生长异常、健康风险)
- **运营效率**提升(料肉比、巡检质量、预警响应)
- **成本优化**空间(饲料成本、服务费用、健康损失)
