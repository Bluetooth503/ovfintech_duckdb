## 📊 资金业务主题域划分

### 一、客户/会员域

管理个人和企业客户的基础信息，是所有金融业务的核心参与方。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `mem_member` | 个人会员表 | name, idNo, mobile, gender, birthday |
| `mem_company` | 企业会员表 | comp_name, short_name, idNo, corporation |
| `fund_member_customer` | 客户信息表 | member_id, member_type, status, business_no |
| `fund_customer_input` | 客户录入信息 | - |
| `fund_create_corp_customer` | 企业客户创建 | - |
| `fund_quality_customer` | 优质客户表 | - |

#### 业务价值

- **客户分类**: 区分个人客户(1)和企业客户(2)
- **实名认证**: 统一社会信用代码/身份证号管理
- **客户状态流**: 创建→待提交→待授信→已授信→待签约→已完成
- **客户评级**: 优质客户标识、风险等级

#### 客户维度属性

- 个人客户: 姓名、身份证、手机号、性别、出生日期、地址
- 企业客户: 企业名称、统一信用代码、法人、联系电话、所属区域

---

### 二、授信/额度域

客户授信额度的申请、审批与管理，是资金贷出前的风控核心。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `fund_member_credit` | 授信额度表 | credit_apply_id, credit_quota, remain_quota, status |
| `fund_member_credit_apply` | 授信申请表 | member_id, apply_quota, apply_period, financial_product_id |
| `fund_create_approval` | 授信批复表 | reply_no, reply_amt, rply_bal, avl_bal |
| `fund_member_config` | 会员授信配置 | manager_no, loop_flag, rate_type, guarantee_way |
| `loan_category_association` | 贷款品类关联 | - |

#### 业务流程

```
授信申请 → 银行审批 → 授信批复 → 额度激活 → 额度使用 → 额度循环
```

#### 关键指标

- **授信额度**: 银行批准的总授信金额
- **可用额度**: 剩余可使用的授信额度
- **剩余敞口**: remaining_exposure - 风险暴露金额
- **授信期限**: credit_begin_day ~ credit_end_day
- **循环标志**: loop_flag - 是否循环授信
- **利率参数**: base_rate + rate_float

---

### 三、借贷/放款域

基于授信额度的放款申请与执行，是资金流出的核心环节。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `fund_debt_apply` | 放款申请表 | contract_code, apply_quota, serial_no, expire_time |
| `fund_debt` | 借据信息表 | loan_no, loan_quota, loan_balance, interest_rate |
| `fund_debt_sync` | 借据同步表 | - |
| `dbt_info` | 借据详情表 | dbt_no, dbt_amt, dbt_bal, int_rate, overdue_days |
| `ctr_info` | 合同信息表 | ctr_no, ctr_amt, ctr_bal, ctr_exp_dt |
| `fund_billing` | 放款账单表 | bill_type, bill_quota, loan_no, serial_no |

#### 业务流程

```
放款申请 → 银行审批 → 生成借据 → 放款执行 → 账务处理
```

#### 放款模式

- **自主支付**: 放款到客户账户
- **受托支付**: 放款直接支付给交易对手
- **多受托支付**: 支持多个受托账号

#### 关键指标

- **借据号**: loan_no - 借款唯一标识
- **放款金额**: loan_quota / dbt_amt
- **借据余额**: loan_balance / dbt_bal - 未还本金
- **执行利率**: interest_rate / use_rate
- **到期日**: expire_time / dbt_exp_dt
- **逾期天数**: overdue_days - 风险监控指标

---

### 四、还款域

借款的还款计划与执行，跟踪资金回流情况。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `fund_repay` | 还款计划表 | loan_no, rate_begin_date, rate_end_date, rate, is_pay_off |
| `fund_repay_detail` | 还款明细表 | - |
| `fund_early_repay` | 提前还款表 | repay_apply_id, repay_quota, pay_off, repay_rate |
| `fund_early_repay_apply` | 提前还款申请 | - |
| `rpy_info` | 还款信息表 | dbt_no, act_rpy_tot_amt, issued_principle, issued_normal_interest |

#### 还款类型

- **正常还款**: 按计划还本付息
- **提前还款**: 提前结清或部分还款
- **逾期还款**: 逾期后还本付息罚息

#### 还款构成

- 本金: issued_principle / act_rpy_tot_amt
- 正常利息: issued_normal_interest
- 拖欠利息: issued_overdue_interest
- 罚息: issued_penalty_interest
- 费用: issued_fee

#### 关键指标

- **是否结清**: is_pay_off / pay_off
- **还款金额**: repay_quota
- **还款利息**: repay_rate
- **逾期天数**: over_due_days

### 五、票据业务域

银行承兑汇票的全生命周期管理，包括出票、背书、贴现等。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `fund_bank_note` | 票据基本信息表 | bank_note_type, bank_note_amt, issu_dt, drft_exp_dt |
| `fund_drawer` | 出票人信息表 | drwr_acct_no, dwr_full_nm, drwr_open_acct_bnk_no |
| `fund_payee` | 收款人信息表 | pye_acct_no, pye_acct_nm, pye_open_acct_bnk_no |
| `fund_bill_endorse` | 票据背书表 | endorse_date, endorse_amt, endorser_member_id, endorsee_member_id |
| `fund_discount_info` | 贴现信息表 | discount_date, discount_interest_rate, discount_amt |
| `fund_add_credit_info` | 保证增信表 | guarantee_type, guarantee_bnk_no |
| `fund_loan_bank_note` | 贷款票据关联 | - |

#### 业务流程

```
出票登记 → 背书转让 → 贴现融资 → 到期兑付
```

#### 票据类型

- **票据类型**: 银行承兑汇票、商业承兑汇票
- **票据种类**: 电票、纸票
- **流通标志**: 可转让/不得转让

#### 业务操作

- **背书**: endorse - 票据权利转让
- **贴现**: discount - 票据融资
- **保证**: add_credit - 增强票据信用

#### 关键指标

- **票面金额**: bank_note_amt
- **出票日期**: issu_dt
- **票据到期**: drft_exp_dt
- **贴现利率**: discount_interest_rate
- **背书状态**: sign_status
- **增信状态**: guarantee_status, add_credit_status

### 六、账户/资金域

客户各类银行账户的管理，用于资金收付。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `fund_bank_account` | 本方银行账户 | acct_no, acct_nm, bank_no, bank_name, acct_balance, avl_balance |
| `fund_payment_account` | 支付账户 | payment_acct_no, payment_acct_nm, payment_bank_name |
| `fund_receive_account` | 收款账户 | receiving_account, receiving_account_name, receiving_bank_name |
| `fund_member_order_account` | 订单账户 | - |
| `fund_member_order_account_detail` | 订单账户明细 | - |
| `fund_member_config_bank_account` | 会员配置账户 | - |
| `account_sign_status` | 账户签约状态 | account_no, sign_status |

#### 账户类型

- **本方账户**: 客户在银行开立的账户
- **支付账户**: 专用于支付的资金账户
- **收款账户**: 专用于收款的账户

#### 账户属性

- 账号、户名
- 开户行行号、开户行名称
- 账户余额、可用余额

#### 关键指标

- **账户余额**: acct_balance / crn_acct_bal - 当前账户总余额
- **可用余额**: avl_balance - 可用于支付的余额
- **签约状态**: binding_status, pay_sign_status

### 七、支付/结算域

资金支付的执行与流水记录。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `fund_online_payment_flow` | 线上支付流水 | payer_acct, payee_acct, tran_amt, tran_type, result |
| `fund_transfer_records` | 转账记录 | trn_ot_cst_acct_no, trn_int_cst_acct_no, txn_amt |
| `freeze_bills` | 冻结单 | freeze_account_no, amount, status, freeze_time, unfreeze_time |
| `fund_member_order_transaction_detail` | 交易明细 | txn_type_id, txn_amount, txn_counterparty_info |

#### 业务流程

```
支付申请 → 资金冻结 → 支付执行 → 流水记录 → 资金解冻
```

#### 交易类型

- **借款**: 资金从资金方流向客户
- **还款**: 资金从客户流向资金方
- **贴现**: 票据贴现资金发放
- **受托支付**: 代客户支付给交易对手

#### 冻结状态

- 0: 冻结中
- 1: 冻结成功
- 2: 冻结失败
- 3: 解冻成功
- 4: 解冻失败
- 5: 已取消

#### 关键指标

- **交易金额**: tran_amt / txn_amount
- **交易对手**: txn_counterparty_info / payee_acct
- **交易结果**: result - 0失败/1成功/2处理中
- **交易流水号**: tran_seq / serial_no

## 🎯 核心业务闭环

### 授信放款闭环

```
客户申请 → 授信审批 → 额度批复 → 放款申请 → 借据生成 → 资金发放
```

### 资金回收闭环

```
还款计划 → 还款执行 → 本息回收 → 借据核销 → 额度恢复
```

### 票据业务闭环

```
出票登记 → 背书流转 → 贴现融资 → 增信保证 → 到期兑付
```

### 风险监控闭环

```
贷后检查 → 风险发现 → 报告生成 → 银行反馈 → 处理跟踪 → 风险化解
```

## 🔍 业务架构特点

### 1. 产融结合模式

- **订单融资**: 基于真实贸易背景的融资服务
- **预付/应收**: 覆盖采购端和销售端的融资需求
- **票据支持**: 银行承兑汇票作为支付和融资工具

### 2. 银企直联

- **系统对接**: 与银行业务系统直连，实时交互
- **多资金方**: 支持多个资金方接入
- **产品配置**: 灵活的金融产品配置

### 3. 全流程风控

- **授信审批**: 严格的授信审批流程
- **贷后管理**: 定期贷后检查和风险报告
- **风险预警**: 实时风险监控和预警机制

### 4. 多元化融资工具

- **流贷**: 传统流动资金贷款
- **票据**: 银行承兑汇票融资
- **订单融资**: 基于贸易背景的融资

### 5. 灵活的还款方式

- **按期还款**: 定期还本付息
- **提前还款**: 支持提前结清
- **循环额度**: 随借随还

## 📝 总结

资金业务是一个典型的**产融结合 + 银企直联 + 多元融资**的金融业务系统：

1. **产品多元化**: 流贷、票据、订单融资等多种融资工具
2. **渠道线上化**: 银企直联，业务全流程线上办理
3. **风控全程化**: 授信审批、贷后管理、风险预警全流程风控
4. **数据实时化**: 账户余额、借据状态等关键数据实时同步

建议在数据仓库建设时，重点突出：
- **授信效率**分析（申请→批复时效）
- **资金成本**分析（利率水平、利差分析）
- **资产质量**分析（逾期率、不良率、迁徙率）
- **产品贡献**分析（各产品的收益与风险）
- **客户价值**分析（客户贡献度、风险等级）
