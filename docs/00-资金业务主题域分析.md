## 📊 资金业务主题域划分（Kimball 维度建模）

### 一、客户会员域 (Customer/Member) — 客户维度

管理客户主数据与基础信息，为授信、放款、还款提供客户维度支撑。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `mem_member` | 个人用户信息冗余 | user_id, name, mobile, idNo, idType, nature, gender, birthday |
| `mem_company` | 机构用户信息冗余 | comp_name, short_name, idNo(统一社会信用代码), corporation, nature, type |
| `fund_member_customer` | 会员订单客户 | order_no, member_id, member_type, financial_product_id, status, business_no |
| `fund_quality_customer` | 资金方推荐客户 | fund_org_id, customer_id, customer_type(1个人2企业), suggest_time |
| `fund_create_corp_customer` | 对公创建客户 | member_id, serial_no, fund_org_id, fund_customer_manager_no, business_no |
| `fund_customer_input` | 进件表 | member_id, fund_org_id, market_id, apply_no, status(0失败1成功2进件中), input_type |
| `fund_member_config` | 会员信贷配置 | member_id, financial_product_id, rate, guarantee_way, customer_no, is_enabled |
| `fund_member_config_bank_account` | 会员配置银行账户 | account_no, account_name, account_bank |
| `fund_member_config_guarantor` | 会员配置担保人 | guarantor, guarantor_idnt_type, guarantor_idnt_no |
| `fund_zd_push_member` | 推送客户表 | push_member_list_attach, fund_org_name, push_status |
| `fund_zd_push_member_list` | 推送客户名单表 | member_id, member_card, member_name |
| `fund_zd_registe` | 客户中登记录表 | registe_time, deal_type, guarantee, registe_type, registe_no, registe_due_date |
| `fund_zd_registe_attach` | 客户中登记录附件表 | attach_name, attach_type(1登记文件2关联文件) |

#### 业务价值

- **客户分层**: 个人/企业用户分类管理
- **进件管理**: 客户入驻资金方的全流程登记
- **中登登记**: 动产融资统一登记系统对接
- **推荐机制**: 资金方推荐优质客户

### 二、授信合同域 (Credit & Contract) — 授信维度

管理客户授信额度审批与合同签约，是放款业务的前置条件。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `ctr_info` | 合同信息(外部系统) | ctr_no, ctr_amt, ctr_bal, ctr_itt_dt, ctr_exp_dt, int_rate, repymt_mth, gry_mth, ctr_status |
| `fund_member_credit` | 授信记录 | credit_quota, remain_quota, customer_contract_no, status(0未激活1激活2作废), credit_begin_day, credit_end_day, reply_no |
| `fund_member_credit_apply` | 授信申请 | member_id, financial_product_id, fund_org_id, apply_quota, apply_period, business_no, bank_quota, status |
| `fund_member_business_apply` | 业务申请 | member_id, apply_begin_date, apply_end_date, apply_quota, guarantee_contract_no, repay_type |
| `fund_member_business` | 业务结果 | business_apply_id, reply_no, result(1成功0失败) |
| `fund_create_approval` | 批复信息 | reply_no, reply_amt, rply_bal, avl_bal, audt_dt, end_dt, gry_mth, bsn_knd, reply_st |
| `fund_create_approval_detail` | 批复明细 | bsn_tp, dtl_amt, dtl_bal, exec_int_rate, dtl_avl_amt, repymt_mth |
| `fund_member_contract` | 合同文件 | serial_no, name, template_name, contract_code, img_code, is_sign, type(1授信2出账3综合授信), contract_source |
| `fund_member_contract_show` | 只读合同 | serial_no, name, contract_code, type(1授信2出账3综合授信4贷前合同) |

#### 业务流程

```
客户进件 → 授信申请 → 授信审批 → 批复确认 → 合同签约 → 额度激活
```

#### 关键指标

- 授信额度、可用额度、授信使用率
- 授信通过率、授信周期
- 合同签约率、合同状态分布

### 三、放款借据域 (Disbursement & Debt) — 交易事实

管理放款全流程，从申请到借据生成，是资金业务核心交易域。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `dbt_info` | 借据信息(外部系统) | ctr_no, dbt_no, dbt_amt, dbt_bal, dbt_exp_dt, dbt_start_date, use_rate, repayment_way, overdue_days, clear_date, dbt_status, lnd_seq_no |
| `fund_debt` | 放款记录 | debt_apply_id, loan_quota, loan_no, expire_time, loan_begin_date, loan_balance, remain_quota, result, interest_rate |
| `fund_debt_apply` | 放款申请 | contract_code, apply_quota, serial_no, expire_time, entrust_account, apply_free_time, dock_type(0线下1系统) |
| `fund_debt_sync` | 放款同步 | customer_contract_no, sync_type_id, sync_status, loan_quota, loan_no, assets_ids, loan_status |
| `fund_billing` | 记账流水 | bill_type(1借款2还款), bill_quota, bill_time, loan_no, serial_no, bill_interest, loan_balance |
| `loan_category_association` | 借据商品关联 | loan_no, cate_first_id, cate_second_id, spu_id, sku_id |
| `fund_order_push` | 订单推送记录 | 订单号、推送状态、推送时间 |
| `fund_order_push_config` | 订单推送配置 | 推送规则、触发条件 |

#### 业务流程

```
额度检查 → 放款申请 → 受托支付 → 借据生成 → 资金放款 → 记账流水
```

#### 关键指标

- 放款金额、放款笔数、放款成功率
- 借据余额、在贷余额、借据状态分布
- 线下/线上放款占比
- 受托支付金额、免息期分布

### 四、还款域 (Repayment) — 交易事实

管理正常还款、提前还款、逾期管理等还款全流程。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `fund_repay` | 还款记录 | order_no, loan_no, rate_begin_date, rate_end_date, rate(利息), is_pay_off, over_due_days |
| `fund_repay_detail` | 还款明细 | repay_date, repay_capital(本金), repay_rate(正常利息), repay_owe_rate(拖欠利息), repay_forfeit_rate(罚息), repay_seq_no |
| `fund_early_repay` | 提前还款记录 | repay_apply_id, repay_quota, pay_off, repay_rate, remain_quota, result, repay_account |
| `fund_early_repay_apply` | 提前还款申请 | loan_no, serial_no, apply_quota_type(1还款金额2还本金额), apply_quota, status |
| `rpy_info` | 还款信息(外部系统) | dbt_no, clr_flg(0指定金额1提前结清2结清当前期), act_rpy_tot_amt, issued_principle, repayment_channel, compensato_symbol |

#### 业务流程

```
还款申请 → 还款清算 → 本金/利息/罚息拆分 → 借据余额更新 → 结清判断
```

#### 关键指标

- 还款金额(本金/利息/罚息)、还款笔数
- 提前还款金额、提前还款率
- 结清率、逾期率、逾期天数
- 代偿金额、代偿笔数

### 五、票据业务域 (Bill/Draft) — 交易事实

供应链票据融资业务，覆盖票据全生命周期管理（出票、背书、贴现、增信、质押出账）。

#### 核心实体

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `fund_bank_note` | **票据基本信息表(核心)** | bank_note_type(银票/商票), bank_note_kind(电子汇票/纸质), issu_dt, drft_exp_dt, bank_note_amt, jx_note_no, status, fbd_endsmt_flg, credit_status, discount(是否贴现), sign(是否签收), bill_src_type |
| `fund_drawer` | 出票人信息表 | member_id, drwr_acct_no, dwr_full_nm, drwr_open_acct_bnk_no, acpter_nm, acpter_opn_acct_bnk_no |
| `fund_payee` | 收款人信息表 | member_id, pye_acct_no, pye_acct_nm, pye_open_acct_bnk_no |
| `fund_bill_add_credit` | 票据增信表 | credit_contract_no, balance, apply_amount, margin_and_ratio, credit_status(0待确认1已确认2已完成3已拒绝), add_credit_role(1出票人2收票人) |
| `fund_add_credit_info` | 保证增信信息表 | acct_no, full_nm, id_no(统一信用代码), guarantee_type(1出票保证2承兑保证3背书保证) |
| `fund_bill_endorse` | 票据背书表 | endorse_date, endorse_amt, sign_status(0否1是2拒签), transfer_flag, endorser_member_id/member_name, endorsee_member_id/member_name |
| `fund_loan_bank_note` | 出票结果信息表 | bank_note_id, bank_note_no, hold_bill_id, loan_no, status, margin_ratio, mrgn_amt, bill_range_start/end, bill_pkg_amt |
| `fund_discount_info` | 贴现信息表 | discount_date, discount_interest_rate, discount_interest, discount_amt, discount_status(0未发起1审批中2审批成功), dcn_into_bnk_nm |
| `fund_transfer_records` | 转账记录明细表 | bank_note_no, loan_no, trn_ot_cst_acct_no, trn_int_cst_acct_no, txn_amt, result |

#### 票据业务全流程

```
出票人登记 → 出票承兌 → 票据登记 ──┬──→ 背书转让 → 签收确认
                                    ├──→ 票据增信(保贴/保证) → 质押出账 → 放款
                                    └──→ 贴现申请 → 贴现入账
```

#### 关键指标

- 票据金额、票面利率、票据到期分布
- 背书次数、背书流转路径
- 增信覆盖率、保证金比例
- 贴现利率、贴现利息、贴现金额
- 质押出账成功率、质押保证金比例

## 🎯 核心业务闭环

### 信贷全生命周期闭环

```
客户进件 → 授信审批 → 合同签约 → 额度激活 → 放款出账 → 贷后监控 → 还款结清 → 额度释放
```

### 票据融资闭环

```
出票登记 → 承兑确认 → 增信申请 → 质押出账 → 放款到账 → 还款解押 → 票据背书/贴现
```

### 风险监控闭环

```
数据采集 → 指标计算 → 阈值比较 → 预警触发 → 工单派发 → 处理反馈 → 风险归档
```

| 类别 | 指标 | 说明 |

| **在贷指标** | 在贷余额、授信在贷余额、借据余额 | 风险敞口 |
| **逾期指标** | 逾期天数、逾期金额、逾期率、不良率 | 资产质量 |
| **票据指标** | 票据金额、贴现利率、增信覆盖率、保证金比例 | 票据融资 |
| **风控指标** | 质押率、抵押率超标预警、流动性覆盖率、借据逾期预警 | 风险监控 |
| **贷后指标** | 贷后报告生成率、季度报告完成率 | 贷后合规 |

## 🔍 业务架构特点

### 1. 多资金方管理

- **资金方体系**: 支持多家银行/金融机构同时对接
- **独立配置**: 每家资金方有独立的银行产品、利率、客户经理配置
- **产品差异化**: 不同资金方提供不同的金融产品

### 2. 双重放款模式

- **线上放款**: 系统自动对接银行接口，全流程在线(占主流)
- **线下放款**: 人工审核放款，系统记录放款结果

### 3. 完整的信贷生命周期

```
进件→授信→合同→放款→计息→还款→结清
```

每个环节都有完整的记录和状态追踪。

### 4. 供应链票据融资

- **票据类型**: 银票(银行承兑汇票)、电子汇票
- **增信方式**: 保贴征信、保证征信
- **票据流转**: 出票→背书→签收→质押→贴现

### 5. 受托支付模式

支持放款资金直接受托支付到指定账户，降低资金挪用风险。

### 6. 贷后风控体系

- **季度报告**: 每季度生成贷后报告
- **风险预警**: 借据逾期预警、即将到期预警
- **还款监控**: 还款履约监控、逾期跟踪

## 📝 总结

资金业务是一个典型的**供应链金融 + 信贷管理 + 票据融资**的复合型业务系统：

1. **信贷核心**: 从进件到结清的完整信贷生命周期管理
2. **双模放款**: 线上自动放款与线下人工放款并存
3. **多资金方**: 支持多家银行/金融机构同时接入与独立管理
4. **票据融资**: 银票/商票的签发、背书、增信、质押、贴现全流程
5. **受托支付**: 放款资金定向受托支付，降低挪用风险
6. **风控体系**: 借据逾期预警、抵押率监控、流动性覆盖率计算
7. **贷后管理**: 季度报告生成与推送的合规管理

### 数据仓库建设重点

- **信贷履约分析**: 放款/还款/在贷/逾期的全维度监控
- **授信额度管理**: 额度使用率、剩余额度、授信到期预警
- **票据融资分析**: 票据规模、贴现利率、增信覆盖率
- **跨域风控联动**: 与仓储域的质押率联动、与资产池域的贷款控制比率联动
- **年累计/总累计**: 支持同比/环比分析的关键指标
