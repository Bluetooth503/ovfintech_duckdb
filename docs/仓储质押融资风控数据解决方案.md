# 仓储质押融资风控数据解决方案

> 基于风控部调研报表问题总结 + IoT 业务主题域分析
>
> 编写时间: 2026-04-27
>
> 核心业务: 仓储质押融资（以仓库中的大宗商品作为质押物）

---

## 一、问题重新定义

风控部的本质痛点不是"缺报表"，而是**无法对质押物实现"看得见、数得清、管得住"**：

| 层次 | 现状 | 目标 |
|------|------|------|
| **看得见** | 摄像头掉线无人管、无主摄像头标识、覆盖率不清 | 全仓库实时可视化，设备异常即时告警 |
| **数得清** | 不规则物品靠人工、称重数据孤立、AI盘点结果未闭环 | 多源交叉验证质押物数量 |
| **管得住** | 监管员行为无监控、电子锁常开无告警、出入库无联动 | 全链路行为可追溯，异常自动拦截 |
| **算得准** | SKU价格波动无预警、质押率无法动态评估 | 抵押率×价格波动→实时风险敞口 |

## 二、指标体系设计

按**业务实体**分层，按**风险信号**分级（绿灯/黄灯/红灯）：

```
┌─────────────────────────────────────────────────────────────┐
│                  风控指标体系（5大域 × 3级信号）               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1️⃣ 质押物资产域                                             │
│  ├── 数量指标                                                │
│  │   ├── 质押物在库数量（系统值 vs AI盘点值 vs 过磅值）        │
│  │   ├── 数量偏差率 = |系统值 - AI值| / 系统值                │
│  │   └── 在库天数 / 最大在库天数                              │
│  ├── 价格指标                                                │
│  │   ├── SKU 市场价（外部数据源）                             │
│  │   ├── 价格日波动率                                        │
│  │   ├── 价格累计波动率（7日/30日）                           │
│  │   └── 抵押率 = 质押物价值 / 贷款余额                       │
│  └── 健康指标                                                │
│      ├── 僵尸SKU率 = 近90天无动销SKU数 / 总SKU数              │

│  ├── 覆盖率指标                                              │

│  ├── 在线率指标                                              │
│  │   ├── 设备在线率 = 在线设备数 / 总设备数                    │
│  │   ├── 摄像头掉线持续时长                                  │
│  │   └── 掉线修复时效 = 平均修复时间                           │
│  └── 电子锁指标                                              │
│      ├── 锁常开时长（超过1h即告警）                           │
│      ├── 异常开锁次数                                        │

│                                                             │

│  ├── 监管员在岗指标                                          │
│  │   ├── 日签到/打卡记录                                     │
│  │   ├── 在岗时长                                            │
│  │   └── 仓库驻场覆盖率                                      │
│  ├── 行为异常指标                                            │
│  │   ├── 非工作时段出入库频次                                 │
│  │   ├── 电子锁操作频次异常                                   │
│  │   └── 单人操作无复核次数                                   │
│  └── 交叉验证指标                                            │
│      ├── 监管员上报数量 vs AI盘点数量 偏差率                  │
│      ├── 监管员上报数量 vs 过磅净重 偏差率                    │

│  4️⃣ 出入库运营域                                             │
```

## 三、风控模型设计

### 模型1：质押物数量交叉验证模型（核心）

解决痛点：不规则物品无法测量、监管员上报数据不可信

```
输入数据源：
  ├── WMS 系统库存数       → 系统账面值
  ├── AI 盘点数量          → 视觉估值
  ├── 过磅净重÷单品重量    → 称重推算值
  ├── 道闸出入库记录       → 物流量校验
  └── 监管员上报数         → 人工值

模型逻辑：
  1. 计算各数据源之间的两两偏差率
  2. 权重分配：过磅(0.4) > AI盘点(0.3) > 系统库存(0.2) > 人工上报(0.1)
  3. 加权得到置信数量
  4. 若置信数量 < 质押要求数量 → 触发红色告警

输出：
  ├── 质押物数量置信区间 [lower, upper]
  ├── 置信度评分（0-100）
  └── 偏差来源定位（哪个数据源异常）
```

### 模型2：抵押率动态监控模型

解决痛点：SKU价格大幅波动无法预警，总行关注大宗商品风险

```
输入数据源：
  ├── SKU 市场价格（需接入外部行情源）
  ├── 在库数量 × 市场价 = 当前质押物价值
  ├── 贷款余额
  └── 抵押率阈值设定（按品类差异化）

模型逻辑：
  1. 实时计算：抵押率 = 质押物市值 / 贷款余额
  2. 价格波动监控：
     - 日波动 > 3%        → 黄灯
     - 累计7日波动 > 10%  → 黄灯
     - 抵押率跌破阈值     → 红灯（如设定50%的抵押率跌破）
  3. 影响面分析：波动的SKU涉及哪些客户、多少贷款余额

输出：
  ├── 客户维度抵押率（实时）
  ├── SKU维度价格波动看板
  ├── 预警名单 + 影响金额
  └── 建议补仓金额（抵押率恢复到安全线需要的货值）
```

### 模型3：监管员行为异常检测模型

解决痛点：监管员作弊风险（打开库房运粮食）、缺乏交叉验证手段

```
输入数据源：
  ├── 电子锁开关记录        → 时间戳 + 操作人
  ├── 摄像头移动侦测事件    → 异常活动时段
  ├── 出入库系统单据        → 合法出入库记录
  ├── 道闸通行记录          → 车辆进出
  └── 监管员考勤/在岗记录   → 人员在位状态

模型逻辑（规则引擎 + 统计异常）：
  Rule 1: 电子锁开启 无对应出库单           → 红色
  Rule 2: 非工作时段(22:00-06:00) 锁开启     → 黄色
  Rule 3: 锁常开 > 1小时                     → 红色
  Rule 4: 道闸有车进出 但无系统单据          → 红色
  Rule 5: 监管员不上报 但摄像头检测到异常活动 → 黄色
  Rule 6: 监管员上报数量 vs AI/过磅偏差 > 5% → 黄色

输出：
  ├── 异常行为事件流（带证据链：锁记录+摄像头截图+系统单据）
  ├── 监管员可信度评分
  └── 仓库风险事件时间线
```

### 模型4：客户综合风险评分模型

解决痛点：搜一个客户所有风险信息、风险日报不完整

```
输入：上述3个模型输出 + 基础业务数据

评分维度（建议权重）：
  ├── 质押物安全（35%）
  │   ├── 抵押率水平
  │   ├── 数量交叉验证结果
  │   └── 在库天数异常
  ├── 设备监控（20%）
  │   ├── 摄像头在线率
  │   ├── 电子锁异常次数
  │   └── 主摄像头覆盖
  ├── 运营异常（25%）
  │   ├── 监管员异常行为次数
  │   ├── 无单出入库事件
  │   └── 出入库频率突变
  └── 资金健康（20%）
      ├── 逾期天数
      ├── 授信使用率
      └── 还款及时性

输出：
  ├── 客户风险评分（0-100，越高越安全）
  ├── 风险等级（低/中/高/禁入）
  ├── 风险雷达图
  └── 改善建议（哪个维度需要加强）
```

## 四、交付物形态

不只是一堆数仓表，而是**面向风控业务场景的数据产品**：

| 交付物 | 面向角色 | 更新频率 | 内容 |
|--------|---------|---------|------|
| **风险日报** | 内部管理层 | 日 | 4个模型输出汇总 + 预警清单 + 待办事项 |
| **客户周报**（总行需求） | 银行总行 | 周 | 客户风险评分 + 抵押率变动 + 重大事件 |
| **实时预警推送** | 风控专员 | 实时 | 红色告警即时推送（钉钉/企微） |
| **客户风险360°画像** | 客户经理/风控 | 按需 | 搜客户名 → 全维度风险信息 |
| **监管员绩效看板** | 运营管理 | 周 | 可信度评分 + 异常事件 + 在岗情况 |

## 五、实施路径

```
Phase 1（1-2周）：指标底座
  ├── 梳理现有数据源，确认哪些指标现有数据可算、哪些缺数据源
  ├── 输出《指标数据源映射表》：每个指标的数据来源、计算公式、更新频率
  └── 明确需外部采购/接入的数据（市场行情价、水电费）

Phase 2（2-3周）：核心模型 MVP
  ├── 先用 SQL + 规则引擎 落地模型2（抵押率动态监控）和模型3（监管员行为异常）
  ├── 这两个模型的数据源最完整，可快速出效果
  └── 输出 MVP 版风险日报

Phase 3（3-4周）：模型完善 + 产品化
  ├── 落地模型1（数量交叉验证）和模型4（客户综合评分）
  ├── 客户周报自动生成
  └── 客户风险360°画像页面

Phase 4（持续）：数据闭环
  ├── 接入外部数据源（市场价、水电费）
  ├── AI 图片体积估算能力（不规则物品）
  └── 模型调优（根据风控反馈迭代权重/阈值）
```

## 六、关键差距分析

要落地这个方案，目前**缺什么**：

| 缺失项 | 影响 | 解决方式 |
|--------|------|---------|
| **SKU 市场行情价格** | 抵押率无法动态计算 | 对接大宗商品行情 API（铜/煤/钢/粮） |
| **监管员考勤/在岗数据** | 行为模型缺输入 | 协调运营部提供打卡数据 |
| **主摄像头标识字段** | AI 分析无法聚焦 | 在设备维度表增加 `is_primary` 标记 |
| **水电费数据** | 客户画像缺维度 | 明确数据来源（电力公司直连/客户上传） |
| **不规则物品体积基线** | 交叉验证缺参考值 | 先用首次 AI 盘点结果作为基线，逐步校准 |
